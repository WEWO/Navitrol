Reflector Based Position Initialization

Updated 09.04.2024

Introduction

Navitrol has a feature that the AGV position can be initialized using reflectors in the mapped environment. This feature can be used only with single level (single map/environment file) system.

Requirements

There are no special requirements for the feature other than that the reflectors should have as good of geometry as possible since the position acquisition is based on triangulation.

Basic requirements for triangulation:

The basic requirement: A well-conditioned triangle. A well-conditioned triangle is the one in which the error in angular measurement has a minimum affect on the computed lengths of the sides of triangle.

It is usually a good idea to create an isosceles triangle with the reflectors in the area where it is known that the position would need to be initialized using them.

There is always an error when the angle is determined. When the lengths are determined using the angles then the side lengths also get affected.

Current possibilities to initialize position

Initialize using the startInfo.bin

Initialize manually from different sources.

Initialize using reflectors.

Scenarios where automatic position initialization can be utilized

It can be used in any area that has reflectors. Sometimes it happens that the environment is not good and the AGV momentarily loses position. In this case the position can be automatically initialized using this feature.

It must be safe to keep the AGV driving in such a situation to continue after automatically re-initializing the position.

Feature flow

Requirements

Three reflectors at the minimum are required for using this feature. It is good to have as many reflectors as possible in the area where the position is to be automatically initialized.

As Navitrol creates sets of three reflectors from the available ones. Therefore, it is better to have enough reflectors around the AGV such that it can create three reflector sets at any given location. The geometry or the angles between the reflectors affect the calculation. The angles between all reflectors should be such that they satisfy the following condition: 0 < β < 180 where β is the angle.

Figure 1: Reflector distances and angles to all other reflectors

Setup

To set the feature up. First add reflectors to the environment. Then set the parameters given in the following table.

ParameterDescription
 pos_ini_refl_based_auto_pos_acq_enEnable the automatic position initialisation based on the reflectors seen
 pos_ini_refl_based_co_dist_reflReflector-based position initialization - Distance (cutoff) between the reflectors to consider determining angles between them.
Needs Navitrol restart.
 pos_ini_refl_dist_match_epsDistance difference between calculated map dist and seen ref dist for auto pos initialisation
pos_ini_refl_ang_match_epsAngle difference [deg] between the observed angle and the calculated angle created at a particular reflector point by vectors joing any other two reflectors
pos_ini_refl_auto_repos_enIf 1, Navitrol will reset position whenever it loses position and not if 0.
 pos_ini_refl_try_countMax number of attempts to initialize position; When position is initialized this counter will be reset. (max value = 1000)
pos_ini_refl_matching_interval_msTime interval (in milliseconds) between two consecutive position initialization attempts.
pos_ini_refl_min_confAutomatic reflector position initialization should not be triggered if position confidence is above this threshold.
 pos_ini_refl_number_setsNumber of reflector sets to be checked from the reflectors seen by the scanner. Modify this to reduce the time taken by algorithm and also the computational cost
 log_pos_ini_refl Log reflector-based position initialisation
pos_init_max_wait_timeOnce the solution is calculated and the position is initialized then Navitrol waits for 1.5*pos_init_max_wait_time before it re-initializes the position.

The parameters then should be tuned so that the feature works properly. Most commonly, the parameters pos_ini_refl_dist_match_eps and pos_ini_refl_ang_match_eps need to be tuned.

Constraints and limitations

Worst case scenario is the one where the reflectors are colinear. This means that the angle between the reflectors is 0 or 180 degrees. In this case that position can be initialized on either side of the reflectors if they are equidistant from eachother. In this case, a good practice is to have them at different distances from eachother. This still will help find a unique position.

Figure 2: Equidistant, colinear reflectors give poor results for position initialization

Example parameters

An example set of parameters is given below. They can be used to start with and tuned thereon to see what helps the most.

Parameters
I,pos_ini_refl_based_auto_pos_acq_en, 1
F,pos_ini_refl_based_co_dist_refl, 25.0
F,pos_ini_refl_dist_match_eps, 0.09
F,pos_ini_refl_ang_match_eps, 0.9
I,pos_ini_refl_number_sets, 30
I,log_pos_ini_refl, 1
I,pos_ini_refl_auto_repos_en, 1
I,pos_ini_refl_try_count, 10
I,pos_ini_refl_matching_interval_ms, 1000
