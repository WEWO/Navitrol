NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

3D positioning

Updated 06.02.2024

Overview

Introduction

3D positioning is a feature, which enables the AGV to localize itself in 3D space by using a 3D laser scanner.

Example outdoor environment for 3D positioning

Requirements

  • Hardware
    • AGV IPC
      • Recommended:
        • Nuvo-9531-PoE-IGN
      • Minimum requirements:
        • CPU: i7-12700DTE
        • RAM: DDR4-16GB
        • SSD: 128GB
    • Mapping PC
      • Minimum:
        • Linux Ubuntu 20.04 operating system
          • Other Linux versions migt work as well, as the software is running inside Docker containers
      • Recommendation:
        • CPU: 6 cores, 64 bit processor, AVX or Neon instructions set support with 12 MB cache
        • RAM: 32 GB, DDR4 with transfer rate of 2666 MT/s
        • SSD: 500 GB or more
        • GPU: NVIDIA RTX A1000 (or equivalent)
      • Notes:
        • More powerful CPU reduces the time used for building the map
        • GPU is optional: Having it increases the mapping performance
        • Big amount of disk space is needed mostly for storing the sensor data recording files
    • Operator PC
      • Minimum:
        • Windows 10, 64 bit operating system
          • .Net Framework 4.7.2
          • C++ Redistributable package 2017 x86 version
        • Screen resolution: 1920x1080
        • CPU: 4 logical core, 2.4 GHz+
        • RAM: 8 GB
        • GPU: OpenGL 2.0 support
        • Storage: 120GB
      • Recommendation:
        • Windows 10
        • RAM: 16 GB
  • Devices
    • 3D laser scanner
      • Minimum:
        • Ouster OS1-32 rev6
        • Range over 100 meters is a required for outdoor navigation
        • 32 scan lines are acceptable for structured environments
      • Recommendation:
        • Ouster OS1-64 rev6
        • 64 scan lines are recommended for outdoor positioning
      • Virtual bumper feature:
        • Ouster OS0-64 rev7
        • Rev7 series offers range up to 100m which is requirement for outdoor navigation
        • Detecting obstacles for slowing down / stopping requires a 90 degree vertical field of view (VFOW) and 64 scan lines
    • IMU
      • Either:
        • GIMU (GIM Robotics IMU)
        • Microstarin 3DM-GX-IMU
    • GNSS
      • Either:
        • GNSS unit:
          • Ublox, C102-F9R
        • Survey GNSS multiband antenna:
          • Ardusimple SKU AS-ANT2B-CAL-L1L2-15SMA-00
        • Base station:
          • Emlid, Reach RS2

Related features

  • Features which are supported with 3D positioning
    • Normal position control driving for any kinematics
    • Virtual bumper
      • Speed limitation when obstacles appear on the route
  • Features which are partially supported with 3D positioning
    • Fleet control
      • 3D positioning map must exists on the AGV IPC beforehand, as it is not downloaded / updated automatically
  • Features which are planned to be supported with 3D positioning later
    • Multilevel
    • Obstacle avoidance
    • Features requiring use of (separate) 2D scanner
      • Load & target detection
      • Docking

Dependencies

  • Sensor fusion
  • ROS
  • GIM

Setup

Quick start

  • GIM software & hardware is installed & calibrated
    • Consult Navitec for setting up
  • ROS2 Docker image is installed
    • See Obstacle avoidance V2 for installing Docker and the images

Start

Once both GIM & Navitrol software are installed, configured and calibrated, start all the software by restarting the AGV IPC. For manual software (re)start, check System restart.

Teaching

Open Navitrol Monitor, and

  • Connect it to the AGV
  • Start teaching by selecting Navitrol -> Start Teach
  • Drive the AGV manually around in areas intended to be mapped
    • Navitrol -> Keyboard Control
  • Stop teaching by selecting Navitrol -> Stop Teach

Generating environment and routes

  • Build 3D positioning map with GIM's map editor tool and export files
    • <map_name>.mapdb
    • <map_name>.pcd
  • Upload the <map_name>.mapdb file to /home/navitec/gim/maps/
  • Open Navithor Tools and open/create a project
  • Import <map_name>.pcd pointcloud file from Project settings -> Add pointcloud file
  • Generate environment in Environment
    • Select imported pointcloud file
    • Adjust the z-axis height range for clearer environment visualization (does not have functional effect)
    • Click Generate
  • Draw the routes in Routes
  • Upload to Navitrol from Export | Upload and request Navitrol to take the files into use

How to use

After

  • Setuping both GIM and Navitrol software
  • Teaching, generating and uploading environment files

Initialize the position (e.g. Set Pos from Monitor) and wait few seconds, that errors

  • ERROR_3D_POSITION_NOT_INITIALIZED
  • ERROR_ABS_POS_NO_MEAS_RECEIVED
  • ERROR_ABS_POS_TOO_LARGE_ESTIMATE_DEVIATION

disappear. If not, try initializing the position again to the correct location

After successful position initialization, Navitrol can be used as normally. The 3D scanner measurements are visualized as projected 2D scan view on Navitrol Monitor.

2D projected visualization of the 3D map and 3D scanner data

System restart

To restart both GIM & Navitrol software, either

  • Reboot the IPC/AGV, OR
  • Run the following commands in the IPC
    • /home/navitec/gim/gim-navitec-deploy/stop_robot.sh (stops GIM software)
    • sudo killall Navitrol (stops Navitrol software)
    • /home/navitec/ros_config/scripts/startRos.sh (restarts Navitec's ROS Docker container)
    • /home/navitec/gim/gim-navitec-deploy/start_robot.sh (starts GIM software)
    • /home/navitec/Navitrol (starts Navitrol software)

Troubleshooting

Most common issues

For the following situations, such as

  • Parameters have been changed
  • Some (specific, predefined) errors/warnings do not disappear

check System restart for restarting the software.

Errors and warnings

207 - ERROR_POSITION_CONFIDENCE_LOW

If the error appears often when there is seemingly no reason for it, check that parameters

  • ros_pose_quality_* are set correctly
  • pos_conf_limit is tuned correctly

564 - ERROR_ABS_POS_TOO_LARGE_ESTIMATE_DEVIATION

The obtained absolute positioning pose estimate deviates by more than sf_abspos_max_position_deviation or sf_abspos_max_angle_deviation_deg from the current pose.

617 - WARNING_SPEED_LIMITED

3D positioning system (if not any other speed limit source) has detected an obstacle in the AGV's movement direction. This might limit AGV's speed down to 0, if the AGV's movement is still towards the obstacle.

839 - ERROR_ABS_POS_NO_MEAS_RECEIVED

Navitrol has not received any valid absolute positioning measurement in a time interval of parameter sf_abspos_max_time_since_valid_meas_ms.

854 - ERROR_3D_POSITION_NOT_INITIALIZED

Position has not been initialized successfully. Set position (from Navitrol Monitor -> Set Pos) and wait few seconds. Repeat if error does not disappear.

945 - WARNING_INITIALIZING_3D_POSITION

If this warning does not automatically disappear after few seconds, try initializing the position again
