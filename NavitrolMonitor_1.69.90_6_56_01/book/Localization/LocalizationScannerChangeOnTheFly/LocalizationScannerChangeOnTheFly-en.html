Localization scanner change on the fly

Updated 26.01.2024

Introduction

In certain kind of environments safety scanners or other low mounted scanners provide reasonable amount of data for localization, while in some other situations they do not. In those cases top mounted scanner might be a better solution.

Navitrol has a feature that makes it possible to change the map and localization scanner(s) used on the fly. Namely this is tied to concept of levels but can be used in single level environment as well.

Level in this case is just selector for map and localization scanners in use.

Current limitations

This feature has some quite strict limitations for the time being.

Levels are used as means for changing maps and scanners. This means that only "positioning only" systems are supported as multiple levels are not supported in other systems.

Front scanner has to be defined/enabled for localization on lower level. Front/rear scanner selection used for localization can then be either

  • Front only

  • Front and rear

Upper level scanner

  • Only one upper level scanner can be used for localization and that scanner has to be defined as measurement scanner 0; meaning parameters scanner_meas_xxxxxxx[0].

  • Measurement scanner 0 in such a system can't be used for other purposes (obstacle avoidance, target detection,..)

Configuration

All scanners used are defined and communication is started at program startup. Parameter changes to scanner models, number of scanners used and connection details (IP:port or serialport) can NOT be changed on the fly. This same applies to P&FR2000 settings:

Parameter
I,scan_pef_samples_per_scan, 2400
I,scan_pef_freq, 25
I,scan_pef_display_on, 0

Changes to other parameters mentioned in this document are effective also when changed and reloaded at run time.

Enabling the feature

Localization scanner change on the fly is enabled by parameter scanner_meas_loc_override_level

I,scanner_meas_loc_override_level, -1

  • Default value of -1 means that this feature is not used (front/rear used on all levels)

  • 0 - means forced override -- upper level scanner always used (can be used for teaching)

  • >0 above this level measurement scanner 0 will be used as localization scanner

NOTE: If this feature is enabled without defining proper upper level scanner scanner_meas_xxxxxxx[0], then ERROR appears as soon as level is changed to this level or higher.

Configuring the scanners

Use and configuration of "front" and "rear" scanners is the same as in any Navitrol system. These are effective in levels lower than scanner_meas_loc_override_level.

Upper level scanner shall be defined as measurement scanner 0 -- using parameters scanner_meas_xxxxxxx[0].

Parameters and example values

ParameterValue
I,scanner_meas_model[0],3 (default is -1 => disabled)
S,scanner_meas_addr[0],192.168.2.182:80 (IP:port)
F,scanner_meas_x[0],0.0
F,scanner_meas_y[0],0.0
F,scanner_meas_sen_ang_deg[0],0.0
F,scanner_meas_beam_min_ang_deg[0],0.0
F,scanner_meas_beam_max_ang_deg[0],360.0
I,scanner_meas_use_nth_meas[0],1
I,scanner_meas_upside_down[0],0
F,scanner_meas_scan_delay[0],0.0

# values 0.995 ... 1.005)

I,scanner_meas_offset_mm[0],0
I,scanner_meas_use_nth_meas[0],1 (#Use only every nth meas)

Position confidence

Position confidence is calculated by default using same parameters for each level. It is possible to define the effective values of parameters corr_conf_min_scale_npts, corr_conf_max_scale_npts for override levels separately using parameters:

I,corr\_conf\_min\_scale\_npts\_override

I,corr\_conf\_max\_scale\_npts\_override

Both have a default value 0 which means "not used". Then corr_conf_min_scale_npts, corr_conf_max_scale_npts are used for all levels. If the value of these parameters is anything >0, then that value is used on the override levels. It is also ok to define only one of these, like

I,corr\_conf\_max\_scale\_npts\_override, 125

In such a case corr_conf_min_scale_npts would be used as min parameter for all levels.

Switching localization scanners on the fly

Whenever current level >= scanner_meas_loc_override_level, additional scanner_meas_only_model[0] is used for localization. In such a case

  • meas[0] scanner becomes effectively front scanner

  • rear scanner is disabled.

It is the "AGV Controller" that tells Navitrol the level and therefore the scanner(s) to use for localization.

Example: I,scanner\_meas\_loc\_override\_level, 2

This would result in using basic front/rear scanners on levels 0 and 1. For level 2 and "above" scanner_meas_model[0] would be used for localization.

Communication to all defined scanners is always active regardless of the use case. This enables immediate localization scanner change on the fly with very small extra cost in CPU load.

Teaching

In order to utilize this feature there has to be a separate map for all levels in use. Matching these maps in exactly same coordinates is critical for the change to work smoothly.

"AGV Controller" tells Navitrol the level and therefore the scanner(s) to use for localization as well as for teaching. Teaching for each map must obviously be done using the scanner intended to be used as localization for that level.

Regardless of the level info sent by "AGV Controller", one can also select/force scanners used for localization and teaching by setting:

Use "normal" front/rear scanners => I,scanner_meas_loc_override_level, -1 Use scanner_meas_only_model[0] => I,scanner_meas_loc_override_level, 0
