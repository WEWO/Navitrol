Lighttpd configuration

Default configuration

Edit /etc/lighttpd/lighttpd.conf to following. Please note that different versions of lighttpd require bit different configurations. Best approach is to add missing configurations (leave include_shell as they are) to existing file, instead of replacing the file completely.

server.modules = (
  "mod_access",
  "mod_alias",
  "mod_compress",
  "mod_redirect",
  "mod_fastcgi",
  "mod_auth",
)

server.document-root = "/var/www/html"
server.upload-dirs = ( "/var/cache/lighttpd/uploads" )
server.errorlog = "/var/log/lighttpd/error.log"
server.pid-file = "/var/run/lighttpd.pid"
server.username = "www-data"
server.groupname = "www-data"
server.port = 80

index-file.names = ( "index.php", "index.html", "index.lighttpd.html" )
url.access-deny = ( "~", ".inc" )
static-file.exclude-extensions = ( ".php", ".pl", ".fcgi" )
compress.cache-dir = "/var/cache/lighttpd/compress/"
compress.filetype = ( "application/javascript", "text/css", "text/html", "text/plain" )

# default listening port for IPv6 falls back to the IPv4 port
include_shell "/usr/share/lighttpd/use-ipv6.pl " + server.port
include_shell "/usr/share/lighttpd/create-mime.conf.pl"
include_shell "/usr/share/lighttpd/include-conf-enabled.pl"

fastcgi.server += (
  "remote_control" => (
  "remote_control.fcgi.handler" => (
    "socket" => "/tmp/remote_control.fcgi.socket",
    "check-local" => "disable",
    "bin-path" => "/home/navitec/remote_control.fcgi",
    "max-procs" => 1
  ))
)

Enable authentication

This step is optional.

Create file /home/navitec/.lighttpdpassword and to that file add line with desired password and username separated with column (:):

username:password(MD5 hashed)

Add following lines to /etc/lighttpd/lighttpd.conf:

auth.backend = "htpasswd"

auth.backend.htpasswd.userfile = "/home/navitec/.lighttpdpassword"

auth.require = ( "/" =>
  (
  "method" => "basic",
  "realm" => "Password protected area",
  "require" => "valid-user"
  )
)

Enable HTTPS

This step is optiona.

  1. Run: mkdir /etc/lighttpd/certs/

  2. Add your certificate file (.pem) to just created location. If using self-signed cert, run following commands in created directory:

openssl req -new -x509 -keyout lighttpd.pem -out lighttpd.pem -days 358000 -nodes

Note:

  • Days can be adjusted

  • Using self-signed certificate requires user to manually trust the certificate

  1. Add following lines to the end of /etc/lighttpd/lighttpd.conf file:
$SERVER["socket"] == ":443" {
  protocoll = "https://"
  ssl.engine = "enable"
  ssl.pemfile = "/etc/lighttpd/certs/lighttpd.pem"

} else $HTTP["scheme"] == "http" {
  # In case of unsecure connection, redirect to secure*
  $HTTP["host"] =~ ".*" {
    # redirect to https, port 443:
    url.redirect = (".*" => "https://%0$0")
  }
}

Disabling suspend and problematic services

Run as root. If services do not exist (possibly installed as a part of graphical desktop, and installation of the desktop should be avoided):

systemctl disable NetworkManager.service

systemctl disable ModemManager.service*

Run as root. Disable all suspends and sleeps:

systemctl mask sleep.target suspend.target hibernate.target hybrid-sleep.target
