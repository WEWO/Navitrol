NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

Debian 11 Bullseye Installation

Updated 14.02.2024

Overview

Debian Bullseye is a Linux distribution that can be used on the AGV:s computer (eg. VBOX) to run Navitrol and other related software. This document guides you throuh the process of installing Bullseye using our custom installer. This customized installer makes it easier to set up a correct running environment for Navitrol. There are also a couple of things that you may want to customize after the installation that are also mentioned here.

Introduction

Debian Bullseye was released August 14 2021 and will be supported by the Debian Long Term Support team and Debian Security Team for 5 years. In December 2022 Navitec has made available the customized installer for Bullseye that creates a suitable running environment for Navitrol using a streamlined installation process. Almost all the configuration that was done manually to the previous versions and delivered in a CloneZilla image is now part of the installer itself.

What's new in this version of Debian image?

  • This image includes Docker, which should allow running containerized applications on the AGV. For example ROS based applications can be installed this way.
  • The image also includes 'sudo' which can be used to become root when needed.
  • Web interface uses SSL by default. It also has password protection enabled by default. The username and password are the same as you would use to login through SSH.
  • The release notes for Debian Bullseye are found at https://www.debian.org/releases/bullseye/amd64/release-notes/
  • The kernel version that is installed by the installer is 5.10.0-19-rt. The kernel includes support for CAN devices through SocketCAN and shouldn't require any extra drivers to be installed.

Requirements

  • A computer that can boot from USB and has a hard-drive that will be used for installation. To run the installer, there should be at least a keyboard and monitor connected to the computer. You may also prefer to have a mouse for navigating the installer.
  • An installation medium, typically a USB stick that has at least 2Gb of space. If you prepare the installation media yourself, you also need the ISO image and a computer running the suitable software (eg. Rufus) to make a bootable USB stick from the ISO file.
    • Rufus is available at https://rufus.ie/en/
    • The ISO file can be downloaded from https://www.dropbox.com/s/e11v2ynzt0g88eh/bullseye-installer.iso?dl=0
  • The computer used should satisfy hardware requirements for Navitrol. The Debian installation itself isn't a limiting factor from hardware point of view.

Limitations

  • The installer doesn't install packages for supporting Touch UI. If that feature is required the support packages will have to be manually installed.
  • The installer doesn't install packages for supporting 3D object detection. If that feature is required the support packages will have to be manually installed.
  • To make the setup easier, most of the decisions you normally make when installing Debian have been made for you. This means that:
    • It will pick the installation drive automatically. Do not connect other drives during installation.
    • It will partition disk automatically, so you can't customize that.
    • locale will be en_US.UTF-8

Dependencies

  • Navitrol installer (.deb package) supports installing on Debian Bullseye starting from version 6.42

Set up

Making a bootable media

Take the ISO image and burn it to DVD or USB stick. On Windows, software called Rufus can be used for this task. Select the DD mode and not the ISO mode when writing to USB.

  1. Plug in your USB stick
  2. Start Rufus
  3. Windows will give you a dialog where you should select "Allow this program to make modifications"
  4. Select your USB stick as the "Device"
  5. Select the ISO image by clicking the "SELECT" button under "Boot selection"
  6. The rest of the options can stay at their defaults:
    • Partition scheme: MBR
    • Target system: BIOS or UEFI
    • File system: FAT32
  7. Click "START"
  8. It shows a dialog titled "ISOHybrid image detected" with options "Write in ISO image mode (recommended)" and "Write in DD Image mode". Select "Write in DD Image mode" and press OK
  9. Wait for it to finish before removing the USB stick

Installation

These installation instructions are tested on VBOX, but the installation should be similar on other PCs as well. The specific keys and boot menu may of course be different.

  1. Plug the USB stick to USB2 port on the back of VBOX (the black one, not one of the blue ones). If you use a blue port, the installer will start, but will run into problems in the middle of the installation.
  2. Enter the BIOS on machine startup by pressing the ESC key (may be different on other machines), select the USB stick as the boot device
    • Use arrow keys to browse to the 'boot' tab
    • Select Boot option #1 using the arrow keys
    • Press Enter and select the USB key using arrow keys, press Enter again
    • Press F4 to "Save and exit"
  3. Boot the machine
  4. The installer should start and ask for keyboard layout, select the desired layout.
  5. The installer should ask for the name of the host. Enter desired name for your AGV.
  6. Installer now runs for a while, but should ask no further questions.
  7. In the end, the installer will warn you that the computer is going to restart. Remove USB stick and press ok.

Fine tuning

Network configuration

To view current network interfaces and their status ip a

Network configuration can be changed with. sudo nano /etc/network/interfaces.d/navitec

The example below are the Navitec default settings. These settings are active if you don't configure anything else.

# The primary network interface
allow-hotplug enp1s0
iface enp1s0 inet static
address 10.10.6.100
netmask 255.255.255.0

allow-hotplug enp2s0
iface enp2s0 inet dhcp

You can also configure a wlan network if needed. This can be in the same file or under a different file, eg. in ´/etc/network/interfaces.d/wlan´

auto wlp3s0
iface wlp3s0 inet static
address x.x.x.x
netmask x.x.x.x
gateway x.x.x.x
wpa-ssid xxxxxxxxxxxxxx
wpa-psk xxxxxxxxxxxxxx

Newer versions of the Bullseye installer provided by Navitec (V3 and up) uses the new Debian naming convention for network interfaces. The new naming convention binds the network names with physical PCI lanes which makes the interface name to physical port correspondence more consistent. With the new interface names you would typically have enp1s0, enp2s0 and wlp3s0 instead of eth0, eth1 and wlan0. To revert the interface names to the old naming style, follow these steps:

  1. Open the following file with a text editor, for example nano /etc/default/grub
  2. Locate the line with GRUB_CMDLINE_LINUX and add net.ifnames=0 biosdevname=0. The line should look like this after modifying GRUB_CMDLINE_LINUX="net.ifnames=0 biosdevname=0"
  3. Update GRUB to use the new configuration update-grub2
  4. Reboot the machine sudo reboot
  5. Now you can check the interface names with ip a

Now you have to change the network interfaces file to correspond with the old names

# The primary network interface 
allow-hotplug eth0
iface eth0 inet static
address 10.10.6.100
netmask 255.255.255.0

allow-hotplug eth1
iface eth1 inet dhcp

You can also configure a wlan network if needed. This can be in the same file or under a different file, eg. in ´/etc/network/interfaces.d/wlan´

auto wlan0
iface wlan0 inet static
address x.x.x.x
netmask x.x.x.x
gateway x.x.x.x
wpa-ssid xxxxxxxxxxxxxx
wpa-psk xxxxxxxxxxxxxx

Setting the time zone

To get the list of time zones give the following command over SSH

timedatectl list-timezones

and then you can select your timezone using this command:

sudo timedatectl set-timezone <your_time_zone>

Lighttpd configuration

The Lighttpd server is configured correctly by default. SSL and password authentication are enabled by default.

Disable authentication

The authentication can be disabled by removing the file ´/etc/lighttpd/conf-enabled/require_password.conf´.

Change the password

Edit file /etc/lighttpd/nt_password. That file should have a line with desired password and username separated with column (:): username:password(MD5 hashed) It is using htpasswd as the backend, so you should create the hash accordingly.

Disable SSL (HTTPS)

You can disable the SSL by removing /etc/lighttpd/conf-enabled/enable-ssl.conf.

How to use

User accounts (SSH)

The installed system will have a user called navitec with the customary default password. The system will allow logins through SSH and will have the default network settings described above. You can use eg. PuTTY to connect to the computer running Debian from a Windows computer, or just ssh from terminal if you are connecting form Linux or MacOS. You can transfer files to and from the computer using SFTP.

Navitrol Updater

Navitrol Updater is a separate piece of software that can be used to install Navitrol on the Debian. It can also be used to send other files. For more information, see Navitrol User Instructions.

Troubleshooting

Installer is not starting at all

  • Ensure that the USB stick is selected as a boot device from the BIOS

Installer encounters a problem (can't find installation media)

  • Ensure that the USB stick was made using DD mode and not ISO mode
  • Ensure that you are using USB2 port to install, the USB3 ports may not work with all firmware
