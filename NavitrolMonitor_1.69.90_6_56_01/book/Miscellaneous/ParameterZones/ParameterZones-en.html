NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

Parameter zones

Updated 26.01.2024

Overview

Navitrol parameter zones can be used to change active parameters of the AGV on the fly based on the AGV's position.

The parameter zones can be used for example to define separate position confidence limits for different areas on the map.

Important note: Before taking this feature into use contact Navitec support to discuss what you are planning to do. This is required because changing some parameters at runtime might cause unwanted side effects and some might not do anything as they are only updated at Navitrol startup.

How to use

Preconditions

  • Minimum software version requirements
    • Navitrol 6.42.00
    • Navithor Tools 5.0.0.0
  • Discuss with Navitec support that this feature will work as intended for your use case.

Basics

Parameter set

  • A set of parameters that will replace the normal ones defined in params.txt
  • Defined using params_zoneX.txt files that must be located in the same folder as Navitrol executable
    • "X" refers to the index of the parameter set. Valid values are 0, 1, 2 and 3.
    • For example "params_zone2.txt"
  • The parameters defined in the parameter sets will override the ones defined in the params.txt
    • This means that only the ones that you want to change should be defined in the params_zoneX.txt
    • Does not alter params.txt file
  • Parameter sets of different indexes do not affect each other in any way

Parameter zone

  • Defines an environment zone and a related parameter set index.
  • Related parameters are taken into use when the AGV enters a zone
    • AGV is considered to enter a zone when its navigation point has been within it for 200ms
  • When the AGV leaves a zone the parameters from params.txt are taken back into use
  • There can be any number of parameter zones

Set up

Define parameter zones

  • Open the Navithor Tools project
  • Select "Zones"-tab from the menu on the left
    • The tab is located near the bottom of the menu
  • From "Zone type"-dropdown menu select "ParameterZone"-option
    • If the option cannot be found, update the software
  • Click on the map to define corners of the desired area for a parameter set
  • From "Zone options"-dropdown menu select the desired parameter set for the created zone
  • For more zones, repeat the previous three steps

Define parameter sets

  • Create a new text file params_zoneX.txt
    • Parameter set indexes does not have to be defined in order. For example, you can have only parameter set of index 2 without defining indexes 0 and 1.
  • Add the parameters that should be changed within zones using that parameter set
    • Uses same format as params.txt
    • Tip: You can copy the parameters you want to change from the params.txt and change their values

Additional info

  • If parameter zone is defined but corresponding params_zoneX.txt file is missing, WARNING_PARAMETER_ZONE_PARAMETERS_NOT_FOUND is set and the parameters from params.txt will be used.
    • In practice this means that the zone has no effect
  • The defined zones can overlap. In case of overlapping zones Navitrol will use the zone with lowest ID defined in tools. see Figure 1
    • NOTE : Zone ID is different than parameter set index (for example ParameterSet_0)

Figure 1: In case of overlapping zones, Navitrol uses lowest zone ID.

  • If parameter set (params_zoneX.txt) is defined but corresponding parameter zone is missing, parameter set is parsed in but never used.

Related features

Parameter set override through supervisor interface

Starting with Navitrol version 6.50 the parameter set can also be selected through supervisor interface message 1035. That selection takes precedence over the parameter zones.

Troubleshooting

I defined the parameters zones but now Navitrol does not start

  1. Check that minimum software version defined in Preconditions section is fulfilled.

AGV enters a parameter zone but nothing seems to happen

  1. Check if WARNING_PARAMETER_ZONE_PARAMETERS_NOT_FOUND is active. If it is, the params_zoneX.txt with the index X that matches the parameter set index of the entered zone (defined in Navithor Tools) is missing. Create the file, save it and reload parameters (or restart Navitrol).

  2. Check that parameter zones are correctly defined in the Navitrol's map file by enabling Navitrol monitor zone visualization:

    • Open Navitrol monitor
    • Connect to the AGV
    • From the top bar select "View"-menu to open a dropdown menu
    • From the dropdown menu select "Visualization Options"-option
    • From "Zone visualization"-box check "Parameter zones"-option
      • If the "Parameter zones"-option is disabled or there is no zones visible when it is checked.
        • Check that the Tools project was correctly modified and successfully uploaded to Navitrol
      • If parameter zones option does not exist
        • Check that minimum software version defined in Preconditions section is fulfilled.

  3. Check that you are trying to change parameters that can be changed on the fly. If you have not contacted Navitec support to discuss your use case, do it now.

Navitrol behavior changes in an unintended manner when entering a parameter zone

  1. Check that you are trying to change parameters that can be changed on the fly. If you have not contacted Navitec support to discuss your use case, do it now.

I have any other issue or the troubleshooting tips did not work for me

  1. Take a log covering the problem situation.
  2. Contact Navitec support and provide the log for further analyzation.
