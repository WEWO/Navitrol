Error Descriptions

Updated 10.09.2024

Error Classification

Navitrol errors are classified into three categories

  • Fatal errors
  • Errors
  • Warnings

Fatal errors and errors stop automatic operation of Navitrol. Depending on the error in question, manual driving using Navitrol may still be possible. Fatal errors differ from normal errors so that they have to be acknowledged by the user before they are cleared. Some fatal errors can't be acknowledged and require Navitrol restart. Fatal errors can be divided into following three categories:

  • Only manual driving is allowed, user acknowledgement is required.
  • No driving is allowed, user acknowledgement is required.
  • No driving is allowed, can't be acknowledged at run time. Navitrol restart is required.

Warnings are set as a notification only. Navitrol operation (like automatic driving) is not stopped in case of a warning. Depending on the warning in question, Navitrol operation can be affected though. Therefore, one should always figure out why a specific warning is set and what it means from Navitrol functional point of view.

Overriding Severity Levels

It is possible to raise the severity level of errors and warnings. In other words, the severity level of Error can be increased to Fatal error, and Warning can be increased to either Error or Fatal error.

To override the severity level of an error or a warning, we need to specify its id and the new severity level. The error/warning id is the numeric value mentioned at the beginning of the title of each errors/warnings (e.g., the error id of ERROR_SCANNER_PROTOCOL_ERROR is 101).

Parameters

NameTypeDefault valueDescription
error_override_id[0...9]Int-1The error or warning id that its severity level needs to be overridden (-1 means no error id).
error_override_level[0...9]Int0New severity level:
0: default severity level
1: warning
2: error
3: fatal error

General Critical Errors

101 ERROR_SCANNER_PROTOCOL_ERROR

Description: Sick S300 scanner protocol differs from 1.2 or 1.3

Error Level: Error

Machine types: All vehicles

Error reason: Navitrol communicates with SICK S300. The scanner reports a communication protocol that is unknown to Navitrol.

How to recover: Contact a system specialists at Navitec or SICK.

102 ERROR_MAP_FILE_READ

Description: The map file could not be read

Error Level: Error

Machine types: All vehicles

Error reason: Something is wrong in the map file (.nte) in the /home/navitec folder. Navitrol is not able to read the file. The file may be corrupted.

How to recover: Try to transfer a new map file to Navitrol. If it does not help contact Navitec.

103 ERROR_MULTIPLE_MAPS_PER_FLOOR

Description: More than one map is found for a floor

Error Level: Error

Machine types: All vehicles

Error reason: There are several map files for the same floor in the /home/navitec folder. Navitrol can only have one map file for each floor.

How to recover: Check the files in the /home/navitec folder. Remove the wrong files.

104 ERROR_TOO_MANY_MAP_FILES

Description: More than supported number of map files

Error Level: Error

Machine types: All vehicles

Error reason: There are more than 200 map files for more than 200 floors in the Navitrol folder.

How to recover: Check the files in the /home/navitec folder. Remove the wrong files.

105 ERROR_PARAM_PARSE

Description: Error parsing parameter file

Error Level: Error

Machine types: All vehicles

Error reason: The parameter file params.txt in the /home/navitec folder is not OK. The file may be corrupted, it may have typing errors (even invisible characters) or wrong number of parameters.

How to recover: Check the error prints in the startlog.txt file. Navitrol tells what is wrong with the parameter file.

106 ERROR_NO_LICENSE_CODE

Description: The license key is invalid

Error Level: Error

Machine types: All vehicles

Error reason: There is no valid license key in the Navitrol IPC.

How to recover: Retrieve a new license key from the Navitrol License Server or contact Navitec.

107 ERROR_THREAD_NON_RESPONSIVE

Description: One or more of the monitored threads in SW is not responsive enough.

Error Level: Error

Machine types: All vehicles

Error reason: CPU overload.

How to recover: SW recovers from this ERROR by itself. This ERROR should be quite exceptional. If it happens though, take a log and contact Navitec for finding a reason for the error.

108 ERROR_DRIVE_CONFIG

Description: Error in drive motor configuration.

Error Level: Fatal Error

Machine types: All vehicles

Error reason: Error in drive motor configuration - like parameters related to drive input/output inversion or scaling.

How to recover: Check and fix all driving related configuration parameters. If reason is not resolved, take a log and contact Navitec.

109 ERROR_STEER_ANGLE_CONFIG

Description: Error in steer angle configuration

Error level: Fatal error

Machine types: Steer

Error reason: Parameters related to input/output inversion or scaling wrong. Steer angle has deviated from the target longer than mot_ctrl_drive_angle_diag_time_ms

How to recover: Check and fix all steering related configuration parameters. If reason is not resolved, take a log and contact Navitec.

110 ERROR_ODOMETRY_SOURCE

Description: This error indicates that the odometry source is not defined preventing Navitrol from positioning.

Error level: Fatal error - requires restart after parameter values have been checked and fixed.

Machine types: All vehicles

Error reason: Parameter odometry_source has not been properly defined.

How to recover: Check and fix related configuration parameter odometry_source.

111 ERROR_PARAMETER_RECONFIGURE_NEEDED

Description: This error indicates that reconfiguration of some parameter file is needed, because parameters used for some specific purpose have changed, and parameter files need to be modified accordingly. This error may be triggered by an issue in any of the parameter files used by Navitrol. Issues triggering this error are rather complex in nature, and are typically related to Navitrol SW updates.

Error level: Error

Machine types: All vehicles

Error reason: Some parameter values need to be redefined or parameter used for some specific thing needs to be changed.

How to recover: Run command .\Navitrol paramreconfigure. This command gives detailed information about changes needed. It can even fix the parameter file(s) automatically in many cases, if given a permission to do so.

Description: Failure in trailer coupling or uncoupling sequence

Error level: Fatal error

Machine types: Omni drive (only when trailer transport feature is enabled)

Error reason:

  • Trailer was lost or connected unexpectedly
    • While coupled or uncoupled the trailer coupling status from supervisor is changed unexpectedly.
    • For example, while Navitrol thinks that the trailer is coupled to the AGV and driving and coupling status changes to uncoupled.
    • No error is raised on unexpected changes during coupling and uncoupling sequences
  • Kingpin was not found
    • During coupling when reversing to kingpin the maximum drive distance is reached without coupling status indicating that the kingpin is detected.
    • Max drive distance is determined by trailer_kingpin_straight_drive_dist
  • The AGV system does not respond to coupling action requests
    • During coupling and coupling sequence the state has been same for too long.
    • Max wait time is determined by trailer_error_timer_ms
  • During uncoupling if the automatic return to route fails
    • Either return to route parameters have not been calibrated correctly or the AGV is too far from the route to return.

How to recover: User should 1. check the situation. 2. correct the cause of the issue 3. send correct coupling status to Navitrol 4. reset the fatal error. After resetting the error Navitrol will initialize trailer transport state based on the given coupling status.

Error level: Warning

Machine types: Those omni drives where Navitrol have control over every limited omni wheel.
