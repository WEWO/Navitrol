Parameter reconfiguration

Updated 19.07.2024

Overview

Sometimes parameters used by Navitrol for some specific purpose need to be changed when Navitrol software is updated. There are numerous possible reasons for this, some examples being

  • Unifying parameter usage
  • Replacing some previously used machine types with new ones

At start-up, Navitrol checks any need for such parameter configuration. If reconfiguration is needed, error ERROR_PARAMETER_RECONFIGURE_NEEDED is raised, and parameter reconfiguration is required before Navitrol can be run again. This error may be triggered by an issue in any of the parameter files used by Navitrol.

Navitrol provides command line utility to check for needed parameter changes. This utility lists all the changes needed to console and stores the same information to a log file .\params_backup\params_reconfigure_log.txt. This utility is even able to fix the parameter files automatically in many cases, if given a permission to do so.

Important note: Parameter changes needed may be related to basic driving. If that is the case, it is recommended to verify updated parameters and all driving directions having AGV lifted off the ground.

Reconfiguration policy

Parameter reconfiguration rules are called reconfiguration policies. Each Navitrol version may include any number of reconfiguration policies, and each policy has different rules for changing parameters.

Navitrol 6.54 has one reconfiguration defined policy, RoboteqDistScale.

How to use

Preconditions

  • Minimum software version requirements
    • Navitrol 6.54.00

Basics

If error ERROR_PARAMETER_RECONFIGURE_NEEDED is raised, parameters need to be reconfigured.

Issue cmd

./Navitrol paramsreconfigure

Navitrol examines all available parameter files and reports whether

  • No reconfiguration is needed.
  • Manual reconfiguration is needed; this may be necessary for different reasons, Navitrol listing the issues why automatic reconfiguration is not possible.
  • Automatic reconfiguration is possible; parameter changes are listed and then user given the possibility to either allow or deny the writing of the updated information to parameter file.

Reconfiguration rules may be different to main parameter file params.txt and override files (params_override.txt, params_zone*.txt).

NOTE: Navitrol updating the parameter file means that any manually added user comments and parameter regrouping is lost. That is why user is always given the option to make the suggested parameter changes manually.

Reconfiguration policies

Reconfiguration policy RoboteqDistScale

This reconfiguration policy checks parameters used for scaling speed as reported by Roboteq motor controllers. Parameter mot_ctrl_drive_rpm_to_m_per_s should be used for that purpose, but some old machine types have used distance scaling parameters instead. Starting from Navitrol software version 6.54, mot_ctrl_drive_rpm_to_m_per_s is used for all machine types.

Parameter mot_ctrl_drive_rpm_to_m_per_s has been mistakenly obligatory parameter for many machine types that have not used it all. If parameter file contains both old style dist_scale parameters and mot_ctrl_drive_rpm_to_m_per_s, automatic reconfiguration is not possible. In such a case reconfiguration suggests manual configuration, and reports what prevents automatic configuration:

ISSUE: Both mot_ctrl_left_dist_scale/mot_ctrl_right_dist_scale and mot_ctrl_drive_rpm_to_m_per_s defined

If one knows that dist_scale parameters have right values, it is possible to overcome this issue by simply removing mot_ctrl_drive_rpm_to_m_per_s from parameter file.

NOTE: Inversion parameters are modified by this policy only for "main" parameter file params.txt. As Navitrol version 6.52 supported only open loop control with Roboteq motor controllers, output inversion parameters are never modified.

Parameter backup files

Whenever any of the current parameter files is modified by reconfiguration, backup file is stored to folder ./params_backup. File name of each such a backup file:

  • starts with original file name
  • includes reconfigure to indicate that backup was created by reconfiguration
  • includes date and time

Backup files are named like:

  • params_reconfigure_backup_2024.04.16_10.50.27.txt
  • params_override_reconfigure_backup_2024.04.16_11.48.37.txt
  • params_zone0_reconfigure_backup_2024.04.16_14.21.30.txt

Console output and logging

Console output and information stored to log by command ./Navitrol paramsreconfigure are almost identical. Some additional information is stored to log file, including Navitrol software version used, date and time.
