Overview

SSL support is currently implemented in Navitrol-Monitor, Navitrol-Navithor and Navithor-MES communication interfaces. It requires the support to be enabled by parameter on each end and the correct certificates to be installed. If only one end is configured to use encryption, connection can't be made.

The use of SSL aims at both encrypting the communications, so that it can not be eavesdropped, and identifying the communicating parties, so that unauthorized parties can not give commands or feed false information into the system. For this purpose, both ends are using certificates, signed by a common trusted root certificate. The different parties have a different common name in the certificate, which prevents stolen certificate from being used to communicate in another role in the system (eg. Navitrol certificate can't be used as Navithor).

This document gives instructions on how to create a root certificate and use that to sign the per-role certificates. It also explains how to install the certificate in windows (where it has to be done system-wide) and Linux (where it is local for only Navitrol). Relevant parameters are listed and some tips are given for debugging issues.

Create Root CA (Done once)

Create Root Key

Attention: this is the key used to sign the certificate requests, anyone holding this can sign certificates on your behalf. So keep it in a safe place!

openssl genrsa -out rootCA.key 4096

Create and self sign the Root Certificate

openssl req -x509 -new -nodes -key rootCA.key -sha256 -days 5024 -out rootCA.crt

Create a certificate

This procedure needs to be followed for each server/appliance that needs a trusted certificate from our CA

Create the certificate key

openssl genrsa -out example\_cert.key 2048

Create the signing request (csr)

The certificate signing request is where you specify the details for the certificate you want to generate. This request will be processed by the owner of the Root key (you in this case since you create it earlier) to generate the certificate.

Important: Please mind that while creating the signing request it is important to specify the Common Name providing the name for the service, otherwise the certificate cannot be verified. Navitrol server Common Name (CN) default is "Navitrol"

Openssl will ask you questions about the certificate to generate like the organization details and the Common Name (CN) that is normally the web address you are creating the certificate for. Navitrol instead uses the role of the machine as the common name.

openssl req -new -key example\_cert.key -out example\_cert.csr

Verify the csr's content

openssl req -in example\_cert.csr -noout -text

Generate the certificate using the example_cert csr and key along with the CA Root key

openssl x509 -req -in example\_cert.csr -CA rootCA.crt -CAkey rootCA.key -CAcreateserial -out example\_cert.crt -days 5000 -sha256

Verify the certificate's content

openssl x509 -in example\_cert.crt -text -noout

Quick guide for creating root, client and server cert for Navitrol --- NavitrolMonitor communication

Create a folder /home/navitec/openSSLCerts in Debian.

Go inside this folder and follow the commands below.

  1. Create root key:
    openssl genrsa -out NavitecCA.key 4096

  2. Create and self sign root cert:
    openssl req -x509 -new -nodes -key NavitecCA.key -sha256 -days 5024 -out NavitecCA.crt

  3. Create the certificate private key for Navitrol:
    openssl genrsa -out agv.key 2048

  4. Create the signing csr for Navitrol:
    openssl req -new -sha256 -key agv.key -out agv.csr

  5. Verify the csr's content:
    openssl req -in agv.csr -noout -text

  6. Generate the Navitrol certificate by using csr and private key:
    openssl x509 -req -in agv.csr -CA NavitecCA.crt -CAkey NavitecCA.key -CAcreateserial -out agv.crt -days 5024 -sha256

  7. Verify Navitrol cert content:
    openssl x509 -in agv.crt -text -noout

  8. Create the certificate private key for NavitrolMonitor:
    openssl genrsa -out monitor.key 2048

  9. Create the signing csr for NavitrolMonitor:
    openssl req -new -sha256 -key monitor.key -out monitor.csr

  10. Verify the csr's content:
    openssl req -in monitor.csr -noout -text

  11. Generate the NavitrolMonitor certificate by using csr and private key:
    openssl x509 -req -in monitor.csr -CA NavitecCA.crt -CAkey NavitecCA.key -CAcreateserial -out monitor.crt -days 5024 -sha256

  12. Verify NavitrolMonitor cert content:
    openssl x509 -in monitor.crt -text -noout

  13. Change Navitrolmonitor key + cert .pfx file
    openssl pkcs12 -export -out monitor.pfx -inkey monitor.key -in monitor.crt

  14. Copy root cert and monitor cert (monitor.pfx) to Windows mmc

    1. add root cert (example NavitecCA.crt) under MMC Certificates/Trusted Root Certification Authorities/
    2. add monitor pfx cert (example monitor.pfx) under MMC Certificates/Personal/

Setting up SSL in Navitrol

1. Default setup

If you use the following names for the certificate files and don't set passwords for them, the setup is just these three steps:

  1. Set the ssl_enabled parameter to 1
  2. Copy the certificate and key with "Navitrol" common name on the AGV
  3. Copy the root certificate (but not the key) on the AGV
FileName
Navitrol certificate/home/navitec/agv_certs/agv.crt
Navitrol key/home/navitec/agv_certs/agv.key
The root certificate (that was used to sign the other keys)/home/navitec/agv_certs/root.crt

2. Extra steps if you don't use the default places for files

  1. Set the ssl_agv_certificate_file to the certificate file path
  2. Set the ssl_agv_key_file parameter to the key file path
  3. Set the ssl_ca_file parameter to point to the trusted root certificate

3. Other options

  1. If the key requires a password, set the password in ssl_pem_password parameter

Example parameters:

## Secure communication parameters: Navitrol <-> NavitrolMonitor
I,ssl_enabled, 1
I,ssl_verify_server_cert, 1
I,ssl_verify_client_cert, 0
S,ssl_agv_certificate_file, /home/navitec/agv_certs/agv.crt #Navitrol private certificate
S,ssl_agv_key_file, /home/navitec/agv_certs/agv.key # certificate private key
S,ssl_ca_file, /home/navitec/agv_certs/NavitrolCA.crt # root cert
S,ssl_pem_password, navitrol # agv_certificate_file password

Setting up SSL in Windows

Add root cert

Navitrol root certificate (NavitrolCA.crt) must be added to MMC so that SSL communication works between Navitec products.

  1. Open MMC

  2. Click File Add/Remove Snap-in...

  3. Choose certificates and then click Add

  4. Select "Computer account" and click Next

  5. Select Local computer and click Finish

  6. Press Ok to close "Add or Remove Snap-ins" pop-up window

  7. Now in your MMC view you should have a Certificates section. Navigate to Certificates/Trusted Root Certification Authorities/Certificates

  8. Click More Actions -> All tasks import

  9. Add root Cert (example NavitrolCA.crt)

I. Browse the Correct cert

II. Choose "Place all certificates in the following store"

III. Click finish

Adding a certificate

If two-way authentication is needed/wanted, a client-side certificate is needed also. This is the certificate for NavitrolMonitor or Navithor. MES interface does not require a client certificate to be sent. The certificate needs to be added under MMC Certificates/Personal/. In this example the client-side certificate is named monitor.pfx

  1. Open MMC

  2. Add Certificates Snap-in as Computer account local computer as done in Chapter 1.

  3. Navigate to Certificates/Personal/Certificates

  4. Import new certificate

  5. Browse and choose the .pfx file

  6. Type the certificate password if it had one

Server and Client certificate Common Names (CN)

By default, Navitrol/Server certificate (agvcrt) Common Name is Navitrol and Monitor certificate (monitor.pfx) CommonName is RemoteMonClient.

If these must be changed, do the following:

  1. Go to the folder where you have NavitrolMonitor

  2. Open Parameters.xml

  3. Search <ServerCommonName> and change the Server common name to match the server certificate common name (agv.crt Common name)

  4. Search <ClientCommonName> and change the Client common name to match the client certificate common name (monitor.pfx common name)

ParameterDescriptionDefault value
ServerCommonNameNavitrol certificate common name (must match to it, example agv.crt)Navitrol
ClientCommonNameMonitor side certificate common Name (Must match to it, example monitor.pfx)RemoteMonClient

Enabling SSL on Navithor

Open settings in Navithor Client and set the following parameters in NavithorServer.exe.config to configure the wanted common names of the certificates. Certicates with the names must be placed into the certificate storage as described earlier in section 8. This must be done before server startup.
