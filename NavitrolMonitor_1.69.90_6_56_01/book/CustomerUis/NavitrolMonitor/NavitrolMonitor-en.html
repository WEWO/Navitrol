Navitrol Monitor - User Instructions

Updated 13.09.2024

Introduction

Navitrol Monitor is a Windows application that aims to work as a service and/or control tool for the AGV onboard software Navitrol. It provides useful real-time information about localization, positioning and system status etc. while also giving the user the possibility to directly control the AGV provided that Navitrol is not running in position-only-mode. In addition to this, Navitrol Monitor does also act as the primary log analysing tool and features a playback mode to read in the binary logs generated by Navitrol.

System Requirements

Navitrol Monitors ability to visualize the information sent by Navitrol in real time sets requirements on both software and hardware:

Minimum hardware requirements

  • Screen resolution: 1920x1080

  • CPU: 4 logical cores, 2.4 GHz

  • RAM: 8 GB

  • GPU: OpenGL 2.0 support

  • Storage: 120 GB

Recommended hardware requirements

  • Screen resolution: 1920x1080

  • CPU: 4 logical cores, 3.0 GHz

  • RAM: 16 GB

  • GPU: OpenGL 2.0 support

  • Storage: 120 GB

Software requirements

  • Operating system: Windows 7, Windows 10 or Windows 11

  • Net Library 4.7.1 installed

    • https://www.microsoft.com/net/download/framework

  • C++ Redistributable package 2017 X86 version

    • https://visualstudio.microsoft.com/downloads/

    • https://aka.ms/vs/15/release/VC_redist.x86.exe

Getting Started

Folder Structure and Password Configuration

The Navitrol Monitor executable NavitrolMonitor.exe is found under its root directory together with its configuration file NavitrolMonitor.exe.config and various dll files. On startup Navitrol Monitor will prompt for a password and by default this is set to ntnavitec. Changing the password is done via the UI by pressing *Change...*and entering the new information. In order to disable the password check completely the user has the possibility to change to an empty password. Resetting the password requires the user to delete the ConfigFiles directory that is generated once the application has been started once.

Connecting to Navitrol

Connecting to Navitrol requires that Navitrol and Navitrol Monitor are in the same network and that the firewall has been configured correctly. The user is then able to add machines via the UI by clicking the Edit button from the main toolbar and setting the name and IP/port of the machine of interest. These fields can later be modified.

To establish a connection, first select one of the configured machines from the drop-down list and click Connect. The connection status can then be seen at the bottom of the main window.

User Interface

This section aims to give an instructional overview of the different options and functionality that Navitrol Monitor offers.

Navitrol Monitor Main Menu

Under the main menu of Navitrol Monitor the user is able to find different file handling alternatives, visualization options, data illustrations etc. The different menus will be covered in detail in the following subsections.

File menu

Under the file menu the user has the possibility to perform general file related actions:

  • Open Log -- Used to open and load a binary log file for visual playback.

  • Save Log -- Used to save a binary log that does not contain any text log. Not recommended when sending logs to Navitec Systems for analysis as the text log is often needed.

  • Convert to Teach File -- Converts the binary log to a teach file that Navithor Tools can process.

  • Exit -- Shuts down the application.

View menu

The view menu lets the user manipulate how things are visualized in the main frame of Navitrol Monitor but does also give access to more advanced features such as the State Form:

Options:

  • Follow Vehicle -- Allows the user to follow the vehicle while it is moving through the map.

  • Freeze Monitor Data -- Stores a shade of the current vehicle position and scanner measurements in a lighter color than the real-time data. Useful when calibrating the odometry of the machine.

  • Show Graphs -- Show/hide the right-side graphs in the main window.

  • Show State Form -- Gives the user access to the Navitrol State Form where machine errors and the Navitrol console outputs are shown. Useful during debugging.

  • Visualization Options -- Visualization Options popup window
  • Draw Trajectory speeds -- Enables a color gradient (from red to green, RGB color heatmap) visualization of the calculated trajectory speed.
  • Show Segment IDs -- Visualizes segment IDs and their directions when zoomed close to the route network

Visualization Options

  • Zone Visualization -- Lets the user enable/disable different zone visualizations. Options are enabled if zones are added to the production in NavithorTools project

  • Front scanner -- Front scanner visualization

  • Rear scanner -- Rear scanner visualization

  • Measurement 0 scanner -- Measurement 0 scanner visualization

  • Measurement 1 scanner -- Measurement 1 scanner visualization

  • Large Points -- Large scanner points visualization

  • RSSI Values -- Received Signal Strength Indication visualization. Helpful when identifying reflectors in the surroundings

  • Distance to environment -- Visualizes how close individual scanner points are to the environment points. See Scanner point distance to environment visualization

  • Dynamic Map -- Lets the user toggle the Dynamic mapping scanner points in the main view.

  • External localization -- Lets the user toggle External localization and DMC Docking V2 visualization

  • Custom properties -- Lets the user toggle Custom properties visualization. See Navithor Tools documentation for further info on Custom properties

  • PLC warnings -- Lets the user toggle PLC warnings visualization

  • Position control triggers -- Lets the user toggle the position control triggers being displayed in the top left corner of the main window. Gives the user information about what causes for example soft stops/hard stops on the position control level. Only available when Navitrol controls the driving.

Charts menu

The charts menu gives access to informative charts that visualize the raw data collected by Navitrol in a more user-friendly way. From here the user has the possibility to retrieve information about odometry measurements, speed and angle reference values, scanner corrections etc. Most of these charts are machine type specific as it is based on the underlying kinematic. Therefore, not all charts will display any information for the connected machine.

Charts:

  • Steer Drive Motors -- Contains information about the steering PID control and the driving PD control as well as measured steering angle/drive speed and the corresponding reference values set by Navitrol.

  • Differential Drive Motors -- Contains information about the drive motor PD control for each differential wheel as well as measured speed vs. reference speed.

  • Mecanum Drive Motors PD Control -- Contains information about the drive motor PD control for each mecanum wheel.

  • Mecanum Drive Motors Speed Control -- Contains information about the measured speed vs. reference speed for each mecanum wheel.

  • Position Control -- Shows the predicted and measured curvature.

  • Speed Control -- Shows the speed reference from different sources as well as the final speed reference vs. measured value.

  • Navigation Point Velocity Vector -- Shows navigation point lateral, longitudinal and angular speed references and measurements.

  • Omni Steer Wheels Control -- Contains information about each steer wheels reference angle vs. measured angle.

  • Omni Steer Motors PID Control -- Contains information about each steer motors PID control.

  • Omni Drive Wheels Control -- Contains information about each wheels reference speed vs. measured speed.

  • Omni Drive Motors PD Control -- Contains information about each drive motors PD control.

  • Odometry Correction -- Contains information about the individual scanner corrections that are applied to the odometry as well correction averages over time and odometry auto correction. Also contains information about the scanner delay relative to the odometry.

  • Positioning -- Contains information about position confidence as well as the cycle times for each localization scanner, odometry etc.

Navitrol menu

The Navitrol menu gives access to certain control commands that can be sent to Navitrol but also feature specific forms that provide more detailed information and control options for certain applications.

Options:

  • Reload Parameter file -- Commands Navitrol to reload its parameter file params.txt.

  • Reload Environment and Route Data -- Commands Navitrol reload the environment and route data.

  • Keyboard Control -- Enables/disables the keyboard manual control. If enabled the user can control the vehicle using the arrow keys.

  • Start Teach -- Commands Navitrol to start the teaching process.

  • Stop Teach -- Commands Navitrol to stop the teaching process.

  • Store Position to Teach File -- Saves the current vehicle position to the teach file which can later be used to define data matrix lines (QR-code).

  • Start SLAM -- Commands ROS to start building a map from the environment as a background process.

  • Stop SLAM -- Commands ROS to stop building a map from the environment as a background process.

  • Level Up -- Commands Navitrol to change to the level above. Related to productions where several floors/levels are used.

  • Level Down -- Commands Navitrol to change to the level below. Related to production where several floors/levels are used.

  • Data Matrix Docking V2 -- Gives access to the Data Matrix Docking v2 Form where information related DMC Docking V2 is displayed

  • Motor Controller Form -- Gives access to the Motor Controller Form where each motor controller's output can be tested

  • Pallet Detection Form -- Gives access to the Pallet Detection Form where information related to pallet detection and docking is displayed. From here it is also possible to make modifications to related parameters.

  • Generate Partial Views from CAD Form - 3D sensor -- Form to generate partial views for 3D Object detection

  • Battery Manager Overview -- Gives access to the Batter Manager Overview where battery related information such as percentage, voltage and error status is displayed.

Language menu

Navitrol Monitor can be configured to use another language than English which is the default. Currently there is support for English and German, but this will be extended in the future.

More menu

Under More there are miscellaneous options that do not fall under the other categories. Currently the user has the possibility to dump scan data from the selected time span to a .txt-file

Navitrol Monitor Action Bar

The Navitrol Monitor action bar is found over the main map view. It contains some of the most commonly used actions such as position initialization and saving logs as well as useful tools such as the measurement tape. In addition to this it is also possible to give manual drive orders via the Drive Orders form or the Custom Route form.

Save Logs

If there at some point would be a Navitrol related issue that requires further analysis it is important to make sure that there is as much information as possible available. Navitrol implements its own logging logic which contains both .txt-logs and binary playback logs which are stored on the IPC where Navitrol is running (see Logging for more information).

Navitrol Monitor lets the user both take and manage these logs via the Save and download logs form. A log is saved by pressing Save log button.
Press Advanced to view the advanced logging options which are explained in the Logging document.

After a log has been saved it is possible to download it to the PC running Navitrol Monitor by selecting the file from the Download Log Files view and clicking Download button.

Set Pos

Navitrol does not have access to its position in a global reference system in most cases. Therefore, it is always necessary to initialize the position of the AGV when it has been moved without Navitrol's knowledge or when the position is lost due to any other reason. This can be done via Navitrol Monitor by first pressing Set Pos and right-click and hold on the approximate location of the AGV and then orient the mouse to select the heading before releasing the right-click button.

Heading adjustment is only active when mouse is placed to point outside of the heading dead-zone (green circle around the AGV). When mouse click-button is released, the position initialization message is sent to Navitrol.

Additionally, in multilevel production areas, we can use level selector to specify the current level of the AGV.

Navitrol will the use its localization algorithm to find the exact position more reliably.

Drive Orders

It is possible to give manual drive orders along the route network via Navitrol monitor and this feature is accessed via the Drive Orders form. If Navitrol is operating in standalone mode this form gives the possibility to operate the AGV in a hold-release manner (note that this requires additional route and Navitrol configurations). Additionally, also for non-standalone Navitrol, it is possible to use the so-called Production Simulator from this form which allows the user to give manual drive orders along the route with several targets and the possibility to loop over the target list.

The simplest drive orders can be given directly by hovering the mouse over the route network and right-clicking at the wished target location. The selected route will be highlighted in green. For more complex drive orders the user has the possibility to Add new drive and then Add targets along the route. If the Loop checkbox is ticked the list of targets will be repeated cyclically. When the drive is defined it can be started from the Drive button and stopped from the Stop button.

Custom Route

Sometimes there is a need to give manual drive commands that are not bound to the route network. This is possible to achieve using a so-called Custom Route where the user can select several nodes in the map by right-clicking that then Navitrol use to create a smooth trajectory. The target speed of this trajectory can be set from the same form.

Measurement tape

The measurement tape functionality lets the user measure distances in the map by left-clicking on the two points of interest.

Protractor

The protractor measures the angle between three points. The range is from 0 to 180 degrees. The first point determines the vertex.

Return to route

When the AGV is about to be driven autonomously after being manually driven it is often the case that Navitrol does not accept the current position since it is too far away from the route network or has the wrong heading. The Return to route tool in Navitrol Monitor sends a request to Navitrol and if the limitations are fulfilled Navitrol will plan a trajectory to a suitable point on the route. See Return to route specific documentation for more details.

Abort

Aborts the current drive.

Visualizations

Vehicle

The wheels of the vehicle are visualized with arrows. The color of the arrow indicates the rotation direction. The wheels are rendered in blue if the measured speed of wheel indicates that the wheel is not rotating. Green color indicates that wheels are rotating forwards and red indicates that wheels are rotating backwards.

The numbers next to the wheels (A0, A1 and A2, see pictures below) presents the wheel angle relative to the AGV frame.

Scanners

Scanners and measurement data are visualized in Monitor with different colors.

Scanner positions are visualized with circles and their measurement data are rendered with matching colors. In total NavitrolMonitor can show 4 different scanner's data: two localization scanners and two measurement scanners.

Localization Scanners

Front scanner data is shown in green and rear scanner data is displayed in red. Example pictures below represent two different AGV: one that is using only front scanner for localization and one that uses only rear scanner for localization.

Scanner points that are not used in localization, are marked with grey colour.

Measurement Scanners

Measurement scanners are additional scanners that can be used with localization scanners. These scanners are used additionally to purposes like obstacle avoidance and target detection. The visualization of these scanners is done accordingly: Measurement scanner with index 0 is visualized with orange and the scanner with index 1 with pink.

RSSI Values

If the scanner provides RSSI (Received Signal Strength Indication) data to Navitrol this option enables a color gradient visualization of the scanner points based on their individual RSSI values. Based on this, the user can identify reflectors in the surroundings.

Scanner point distance to environment

Visualizes how close individual scanner points are to the environment points (color gradient from red to green, RGB color heatmap). Scanner point distance visualization is only applied to scanner points that are used for localization and are close enough to the environment points

Large scanner points

Scanner points can be enlarged in the monitor rendering view.

Route Segment ID:s

Segment ID visualization is an advanced tool to observe where segments start and end. In addition, it informs the driving directions of the segment lanes. Visualization is only active when user zooms close enough to the route network and "Show Segment IDs" is toggled on. Example of the visualization below:

Text VisualizationDescription
RightFw: 70Segment ID 70 is the right side forward going lane
LeftBw: 69Segment ID 69 is the left side backwards going lane

Segment ID visualization helps to validate Navithor Tools segment splitting. In addition, the feature can be used to troubleshoot issues that are occurring with some specific segment ID:s.

Binary Log Playback

In addition to visualizing live data streamed by Navitrol, Navitrol Monitor is also able to replay historic data either after being disconnected or from a binary log file saved by Navitrol. Replaying a binary log file is done by dragging and dropping it on Navitrol Monitor's main window.

The available time range can be further restricted by the upper slider in the picture below while the lower slider lets the user select the time of interest.

Level selector

In production areas with multiple levels, a level selector will appear at the bottom-right corner of the screen. This selector allows us to switch between different levels of the production area. The current level of the AGV is indicated by a green shade in the level selector. If the current viewing level in Monitor is different from the AGV's level, then it is shown in cyan. The level IDs corresponding to each level are written on the right side of the level selector.
