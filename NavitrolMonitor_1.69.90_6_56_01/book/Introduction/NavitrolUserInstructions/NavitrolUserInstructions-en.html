Overview

Introduction

Navitrol is a positioning and navigation software for AGVs designed by Navitec Systems. While positioning and navigation are the main features, Navitrol is capable of large variety of AGV control tasks, such as:

  • Target and Obstacle detection
  • Pickup and drop-off functionalities
  • Sensor fusion

AGV executing a navigation and load-handling task

The positioning feature provides the position of an AGV in map coordinates. The positioning is based on dead reckoning data combined with laser scanner corrections. The laser scanner corrections are based on matching received 2D measurements to the taught environment. It is possible to use Navitrol for positioning only applications. In these applications Navitrol is typically used for position correction source by control software running on separate hardware.

In the case when Navitrol is being used to control the AGV directly, the Navigation computer is connected directly to the motor controllers and Navitrol controls the output of the motors or commands are sent to other controlling hardware through the Ethernet Supervisor Interface. The navigation feature provides control references for an AGV to drive along predefined routes combining position control with the positioning.

Navitrol requires an environment model for positioning. This model is created by first teaching the environment by driving the AGV manually around the production area and then using Windows-based Tools software to generate environment and routes. In addition a CAD model of the environment can be used for correcting scaling and misalignment issues.

Navitrol is a Linux software run on an IPC installed on the AGV capable of communicating with various sensors and control devices (See Supported Devices).

Requirements

  • Navitrol software runs on Linux Debian operating system.

    • Supported Debian versions are Buster (10) and Bullseye (11). Recommended version for new installations is Bullseye (11)
    • See Linux Image installation

  • IPC must fulfill the Hardware Requirements

  • Navitrol software needs a license to be run. See Licensing for detailed information.

Set up the IPC Software for Navitrol

Navitrol is run on Linux. Commands are given using a SSH terminal connection over Ethernet. Linux has a Samba file server for transferring files.

Passwords

There are two user accounts that are already set up.

SSH Terminal Connection

The SSH connection is used for giving commands to Linux. A recommended program for taking connection is Putty.

Putty can be downloaded from: chiark.greenend.org.uk/~sgtatham/putty

Set up the IP address to the Host name field and press open to make a connection.

IPC Network connections

The default IP address of the Navigation computer is 10.10.6.100. If the IP address needs to be changed the modification is done in the location /etc/network/interfaces. Network configurations are explained more detail in:

  • Debian 11 Bullseye Installation - Network configuration
  • Debian 10 Buster Installation - Ethernet configurations

After changing the network configuration a reboot is needed.

Transferring Files

The onboard Linux of the Navigation computer runs as a samba server for transferring files between your Windows computer and the IPC.

The connection can be made using Windows explorer.

Setting the address in Windows explorer

After making the connection the user name and password is asked. Enter the user name navitec and password navitrol.

The simplest way to access Samba network drive is to type \\IPC_IP\\navitec\ to Windows Explorer address line, and then type in credentials. As described above. This can however fail in different ways: Windows may fail to connect at all, or the authentication fails because of Windows trying to force specific domain or username. In all those cases it is worth trying to map the network share as a drive. Mapping can be done with File Explorer or with command prompt.

Other option is to use WinSCP to transfer files: https://winscp.net/eng/index.php

Map network drive
  • Right click "This PC" in File Explorer
  • Select "Map network drive..."
  • Add path \\IPC_IP\\navitec\ to Folder, and check "Connect using different credentials"
    • Don't use windows credentials, instead insert username and password matching Navitrol IPC configuration.
Mount with command line
net use DRIVE: \\\\IPC_IP\\navitec /user:USERNAME PASSWORD

Example:

 net use K: \\\\192.168.1.100\\navitec /user:navitec navitrol

Install or Update Navitrol

Since version 6.28.00 Navitrol software is distributed as Debian installation package. There are two alternative ways to handle software installation. User can either use NavitrolUpdater or directly use Debian Package Manager to handle the installation. Use of NavitrolUpdater is the preferred way when updating several AGV's at a time.

NOTE: If you are installing Navitrol the first time, installation is done from command line

Command line

The package name is Navitrol_version_architecture.deb, for example Navitrol_6.28-0_amd64.deb.

Installation:

  1. Copy the installation package to the Navigation computer using SCP or samba

  2. Open SSH connection to Navigation computer

  3. Gain root privileges using 'su -' command.

  4. Install package with dpkg -i Navitrol_6.28-0_amd64.deb command on linux terminal

  5. Navitrol is now installed to run at startup, but it still needs a licence and the parameter file params.txt

    1. Contact Navitec support for the license and base params.txt file

NavitrolUpdater

  1. Choose what do you want to update, any combination can be selected

    • If Navitrol, check "Update Navitrol" and click "Browse" to select used Navitrol installation package (*.deb)

    • If parameters, check "Update params.txt"

    • If override parameters, check "Update params_override.txt"

    • If environment file, check "Upload Environment file"

  2. Choose the files for each checked option with by pressing "Browse" button for each file

  3. Choose the AGVs that you want to update

    • You can add AGVs to list

    • Only the AGVs that where "Update" is checked are updated

  4. Hit the "Update Selected Navitrol Clients"

NavitrolUpdater - Trouble shooting

Updater does not accept the correct password

Reason: The linux system on the IPC does not allow direct login as super user.

Solution: Enable direct login as super user

  1. Open terminal in the IPC running Navitrol.

  2. Login as super user using "su -" command and navitrol as password

  3. Open sshd_config file using nano /etc/ssh/sshd_config command

  4. Add new line to the file: PermitRootLogin yes (the place of the line does not matter)

  5. Save file with ctrl+s and close it with ctrl+x

  6. Restart the ssh process with /etc/init.d/ssh restart command

Configure odometry source

The most common way to provide odometry data is either through CAN bus or Ethernet supervisor interface. Other options currently are EtherCAT and serial.

In most machine types odometry source is selected automatically, and related communication interface enabled automatically as well. When using specific machine types, such as machine_type_id 44xxxx, odometry source must be defined using parameter odometry_source. More detailed information on odometry source selection and configuration can be found from Odometry Source, that document describing also the parameters used to enable other communication interfaces.

Further instructions for setting up and using different communication interfaces:

  • CAN CAN communication
  • Ethernet interface Ethernet Supervisor Interface
  • EtherCAT interface EtherCAT communication
  • Serial bus, with Roboteq motor controllers only Roboteq Controller Configuration

Check also the list of supported devices or ask additional information from Navitrol support. List of supported devices: Supported Devices

Initial Navitrol run

To run Navitrol on the IPC, the following prerequisites have to be done:

  • IPC installed with Debian Buster (10) or Debian Bullsey (11)
  • Navitrol software package (.deb) installed under /home/navitec folder

After this the initial Navitrol parameters are needed. The initial parameter file is provided by Navitec support.

Parameters

Parameters affecting the operation of Navitrol are in the file params.txt in the /home/Navitec folder. It is a text file which can be edited with any editor which can output a plain text file. Parameters that are not listed in params.txt file are used in Navitrol with their default value

A line in the parameter file may be either a comment line or a parameter line. A comment line starts with # and a parameter line with the parameter type indicator F (Float), S (String), I (Integer) or U (Unsigned Integer).

In addition to the parameter type a parameter line consists of the name of the parameter and the current value. Then there is an optional comment about the parameter. A comma is required between each obligatory item of a parameter definition.

Navitrol reads in the parameter file when it starts. If not being able to parse the parameter file Navitrol does not start but writes about the problem on the terminal. A typical error in the parameter file could be a value outside the minimum and maximum limits. If the real parameter value needs to be set outside the minimum or maximum then the limits must be changed accordingly.

./Navitrol

After a successful test run (Navitrol starts running on terminal), proceed to the Initial calibration Guide.

Troubleshooting

Most Common issues

Navitrol does not run with command ./Navitrol

  • Check that you have installed Navitrol to /home/navitec: 
dpkg -s navitrol
  • Check that you have working parameters that Navitrol can parse. Navitec Support will provide you the parameters

Cannot change parameters from params.txt

params.txt file might need permissions for reading and writing. The permissions can be added with linux command:

cd /home/navitec
chmod +rw params.txt

Logging Functions

Navitrol provides a list of commands that can be used in order to write specific log messages to files and/or view them in real time. Amount of log lines and the threads of interest can be also specified manually. Loggings functionalities are explained on their own document Logging
