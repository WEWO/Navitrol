Teaching Instructions

Updated 26.01.2024

Overview

Introduction

Teaching is a process of recording information from the AGV's sensors such as laser scanners and PGV cameras and storing the information in a format that can be used to create an environment model in which the AGV can navigate accurately. In Navitrol the result of teaching is a teach file that can be imported to Navithor Tools software for further refinement. One environment model can be created from a single teach file or by combining the information from multiple teach files. This document describes the best practices and practical hints for teaching and editing production area to generate a map for Navitrol positioning. There are separate documents for GPS positioning and external localization.

Teaching starts with driving the AGV manually through the production area in teaching mode. Teaching is stopped after driving, and teach file is stored on the Navigation computer. Teaching can be repeated as many times as necessary to map the whole production area. After the teach files are generated, Navithor tools software is used to build a map of the production area.

Teaching process is described step by step in the next chapter, and further guidelines for teaching and matching large areas are provided in the later chapters.

In a simple area this procedure is simple and straight forward but when the area gets bigger and more complex, some best practices are needed.

Creating a map from the teach files is called matching. Navithor Tools documentation explains matching in more detail. Still, as teaching and matching are tightly related to each other and the map making process, some matching tips are given at the example at the end of this document.

Requirements

  • Navitrol installed on the AGV
  • Software to control teaching
    • Navitrol Monitor,
    • Navitrol web interface, or
    • Supervisor for starting and stopping the teaching using message 1004 - Start/Stop teaching
  • The area to teach is cleaned up so that the AGV's scanners can see static objects clearly from the path the AGV will be driving. Static objects are for example the walls and pillars in the facility and everything that is bolted down to the floor. If possible, all non static things should be moved away from the area to be taught.
  • AGV must be well calibrated
    • For teaching a calibration / testing area scanner calibration is enough
    • For teaching a production area the AGV must be fully calibrated
    • Take a look at the calibration instructions if you are unsure if the AGV is good enough for use in teaching.

How to use

Teaching steps

  1. Clear the teach area from dynamic environment
  2. Drive the AGV to the desired starting location
  3. Enable teaching mode
    • Teaching mode requires that Navitrol is running.
    • Teaching mode can be turned on using the web interface, Navitrol Monitor or supervisor message 1004.
    • When teaching mode is on Navitrol will show a warning WARNING_TEACH_MODE_ENABLED
  4. Teach the environment by driving the AGV manually
    • Follow Teaching calibration environment or Teaching production environment instructions depending which one you are about to teach
    • Driving can be done using the web interface joystick or the AGV's own joystick. Navitrol Monitor keyboard control should be used only as a last resort if no other means of driving the AGV are available.
  5. Stop teaching
  6. Import the teach files to Tools
    • See, How to find teach files

How to find teach files

Navitrol saves the teach file when teach mode is turned off. All teach files are stored on the IPC in folder /home/navitec/teach and they can be taken from there either using Navithor Tools, SCP client or Samba.

The recommended way to get the teach files is to download them using Navithor Tools. This also automatically adds the teach files into your Tools project.

Teaching calibration environment

The calibration environment is an area where the AGV will be calibrated, tested and fine tuned. Typically such an environment is a relatively small rectangular room with clear straight walls. See Initial Calibration Guide for more information about calibration.

  1. Check that the scanners are calibrated
  2. Clear the area so the scanners can see the walls and corners
  3. Drive the AGV to one end of the room
  4. Start teaching
  5. Drive straight forward (for about 10 meters)
    • It is essential to drive straight as possible to minimize odometry error, as odometry has not been calibrated yet
  6. End teaching

Teaching production environment

The production environment is an area where the AGVs will navigate while performing production orders.

  1. Plan the way the area will be taught. Remember that the information the AGV gathers during teaching will be used later on in the navigation. If you have not taught an area before, read this document carefully before continuing with the teach process. Here is a short list of things to take into consideration:
    • Drive the same way as the AGV will drive when it is executing tasks.
    • Laser scanners have a limited view which is important to understand when teaching.
      • The goal is for each scanner to see as much static environment as possible
    • As a rule of thumb, it is easier to do longer teach drives and have fewer files than have several small teach files.
      • For large production environments, it is recommend to teach one room / corridor at a time as odometry error typically accumulates over long teach drives. This can make matching harder. If you face this issue it is recommended to teach the environment in smaller pieces.
  2. Take the AGV into the starting position for teaching. Good places to start are usually the main aisles that can be driven around the whole warehouse or production facility.
  3. When the AGV is in place facing to the correct direction turn on the teaching mode.
  4. Drive the AGV calmly and as straight as possible according to your plan.
    • Remember that the goal is that the scanners see all static environment necessary for navigation during the teaching.
  5. After you have finished driving either by arriving to a dead end or by arriving to the place where you started turn off the teach mode

Teaching can be done in one or several drives. The number of drives required depends on the availability and complexity of the production area.

Example of teaching warehouse

In this example the warehouse is so large that scanners cannot see from one side of the room to another. Because the scanners cannot see the whole room at once the data must be aligned by matching the same features from multiple teach runs.

Teaching plan and execution

Here is a recommended order to teach the example layout.

Teach outer limits first as a closed loop

When teaching closed loops, it is recommended to teach the area so that beginning and end of the loop are visible in the same teach file.

Then teach the spaces between the racks

Matching the teach data in large room

Start the matching from the outer limits first and then match the inner path to the outer limits.

Matching is usually done with following steps:

  • Matching the closed loop. Quick explanation below.
  • Finalizing the outer limit measurements.
  • Inter matching inner path one aisle at the time.
  • Matching the aisle from ends to the finalized measurements.

Matching the closed loops

Closed loops are matched in following steps:

  • Intermatch the measurements at the whole area
  • Match the beginning of the loop to the CAD (if CAD available)
  • Finalize the measurements at the beginning of the loop
  • Match the end of the loop to the finalized measurements

Troubleshooting

Most common issues

Teach file is not generated

In case you can't find the teach file after teaching make sure that:

  • Teaching was on during the teaching (WARNING_TEACHING_MODE_ENABLED visible in Monitor)
  • Teaching has been stopped.
  • There is enough space on the hard drive.

Automatic matching can not correct the errors from the teach file

This is usually a sign of poor calibration or wheel slipping. To solve this you should teach the area again after verifying the AGVs calibration. During your new teach try to drive slower and minimize turning as turns induce usually the most error in the odometry.

Errors and warnings

305 - WARNING_TEACHING_ALREADY_ON

This warning is briefly set on in case teaching mode is already on when some source tries to set the teaching mode on.

317 - WARNING_TEACH_MODE_ENABLED

This warning is set on when the teaching mode is on.
