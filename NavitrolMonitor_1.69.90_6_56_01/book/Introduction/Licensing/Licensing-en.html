Navitrol Licensing

Updated 11.06.2024

Glossary

TermDescription
FingerprintingA method for uniquely identifying the target where application runs.
Device-locked licenseDevice-locked licensing uses target fingerprinting and locks the license to the target.
Floating licenseFloating license is not locked to any specific target/HW. When a licensed software is started it leases a license from the License Server and frees the license when it shuts down. With Floating license the number of simultaneously running instances is limited by the license.

Overview

Navitrol supports two different licensing methods, Legacy license and Regular license. Legacy licensing model has been used from the early days of Navitrol. It does however have multiple practical limitations, and therefore new licensing method, Regular licensing, has been introduced.

Regular licensing is the recommended future-proof solution. Existing Legacy licenses are still supported, but all new (or future) licensed features are not supported by Legacy licensing method.

Current license details

Detailed information about Navitrol license for a connected Navitrol can be seen in Monitor by selecting "Help-About" from menu.

All this information is also logged in the beginning of each text log file (startlog.txt and log.txt). Examples of logged data are shown in the following chapters.

NOTE: Hardware and license details are different in each case, as they depend on the hardware, licensing model and licensed options in use. Therefore, any license details shown in this document are just examples of one specific case.

Licensing method selection

Licensing method (Regular vs. Legacy) used is selected at run-time by Navitrol. Regular licensing is prioritized, and it is used if license file LicenseDetails.json exists on the system. See Regular license below for details. If there is no LicenseDetails.json file, Navitrol switches to Legacy licensing method for backward compatibility. It is only the existence of this file that matters. If license file exists, but has invalid data or format, then Navitrol still sticks to Regular licensing and raises a related error.

In the beginning of each text log file there is some information about run-time system and licensing method used. The section that shows license information is referred here as a license section.

License models

Regular licensing supports different licensing models. The currently available ones are

  • Offline Device-locked license
  • Floating license

Legacy licensing supports only Offline Device-locked licenses.

Backward and forward compatibility

Navitrol can use any license created for older SW version, assuring full backward compatibility.

Regular licensing provides full forward compatibility as well. Legacy licensing on the other hand has limitations with forward compatibility. It is not guaranteed that license created for a specific Navitrol version would be usable with older Navitrol versions.

Basic license

A basic license is simply a license to run Navitrol without additional licensed options.

License with options

A license with options is a license to run Navitrol with predefined additional licensed options. A license in this case includes information about all enabled licensed options/features.

In Legacy licensing all licensed features are enabled by specific parameters.

Regular licensing uses the same parameters and mechanisms for old license options, but licensed option can also be tied to something else than parameter values (like kinematic type of the machine).

All enabled license options are logged into beginning of the text log files in license section:

Regular license

Regular licensing has several benefits over Legacy licensing from user point of view, like:

  • More license models - supports currently Offline Device-locked license and Floating license. More options available later.
  • All current and future license options/features are supported.
  • Full forward and backward compatibility for licenses created for other SW version.
  • License validation time is quite deterministic, typically instant. Time may vary with Online licensing models (Floating license).
  • Possibility for time limited trial licenses.
  • Possibility for more tailored licensing models and fingerprinting methods in the future.

Activation mode

License models in Regular license concept are called activation modes. The two Regular license activation modes at the moment are:

  • DeviceLocked - Uses Offline Device-locked license. Device-locked license uses target fingerprinting and locks the license to the target.
  • Floating - Floating license is not locked to any specific target/HW. When a licensed software is started it leases a license from the License Server. The License Server allocates a license for that instance (or target) if there is a free license available. Licensed software needs to renew the lease periodically, and finally free the license when it shuts down. This method limits the number of instances that can run at the same time. All licensed software instances need an Internet connection for communicating with the License Server at run time.

License file LicenseDetails.json

A license file in a target system is mandatory for Regular licensing to work. License file is required for all licensing models. This JSON formatted file provides information for license activation and validation, most important ones being the LicenseKey and ActivationMode.

Mandatory license file information from Navitrol point of view include:

  • Product – Shall be NavigationSoftware. Anything else results in license file rejection as license is not meant to be used with Navitrol.
  • LicenseKey – Created by the License Server. License key is used for license activation and validation.
  • ActivationMode – Activation mode to be used by Navitrol (shall be one of the predefined values: DeviceLocked, Floating).

An example license file content having mandatory information only is shown below.

  {
    "LicenseKey": "3A432A-E840E3-42F7AB-A35E2E-6D0317-F69792",
    "ActivationMode" : "DeviceLocked",
    "Product": "NavigationSoftware"
  }

A license file may contain additional information as well. Optional license file information includes:

  • FileGenerationDate - License generation date is typically included, but not regarded as mandatory information by Navitrol.
  • Custom fingerprinting - License can be defined by the License Server to use a specific custom fingerprinting method. In such a case, details are defined in a license file.

How to set up a Regular license

Steps to set up a Regular license are:

  • Order a license from Navitec Systems. Process starts with creating the license in the License Server. Basic information needed in this phase are:
    • License options that shall be included in the license, including machine kinematics.
    • Target system - Navitrol running on a bare HW, a Virtual machine or a Container. NOTE: Navitec Systems may limit the target options available. Trying to activate license in a VM/Container if that is not allowed, results in an error.
    • Other things to decide at this phase are
      • Fingerprinting method - Valid for Node-Locked licenses only.
      • Validity
        • Time - By default a license is perpetual, like in Legacy licensing. There is a possibility to define a limited Validity time.
        • SW versions - By default a license is not version limited. License can be limited up to a specific SW release version only.
      • Possible trial license and trial period
      • + more
  • When detailed license information is known, license can be created in the License Server.
  • License Server provides the license file LicenseDetails.json.
  • License file LicenseDetails.json is copied to target system sub-folder ./license (/home/navitec/license).
  • After that license needs to be activated and validated. Steps required for activation depend on a license activation mode used. See License activation and License validation below for details.

License activation

Unlike in the Legacy licensing method, each license must be activated before being valid. License activation always involves the License Server which keeps track of created and activated licenses.

Prerequisite is the license file LicenseDetails.json existing in target system sub-folder ./license (/home/navitec/license).

Activating Offline Device-locked license

Offline Device-locked license activation is clearly separated from validation. Creating offline request file and offline activation are done only on user request using commands shown below. Running these commands requires root privileges.

Steps are:

  1. Generate offline request file LicenseOfflineRequest.gz file on sub-folder ./license by issuing the command 

    root@navitec:~$ ./Navitrol licreq

    On successful command one should see on the console:

    Packed license request files to: "./license/LicenseOfflineRequest.gz"
DeviceLocked license request file creation SUCCESSFUL

  2. Provide the LicenseOfflineRequest.gz file generated in target HW for license generation.

  3. Offline response file LicenseOfflineResponse.dat is generated on the License Server using offline request file as input.

  4. Offline response file is transferred/copied to Navitrol ./license folder.

  5. Activate license in Navitrol by issuing command 

    root@navitec:~$ ./Navitrol licact

    This command uses the license file and the offline response file as input. On successful activation one should see on the console:

    DeviceLocked license activation SUCCESSFUL

Activating Floating license

Floating license activation (license leasing) is performed automatically at each startup. After successful activation/lease, Navitrol periodically validates and renews the license.

License deactivation

Floating licenses are automatically deactivated by Navitrol at each exit/shut-down, which frees the leased license for someone else to lease. If Navitrol for any reason fails to do that (like communication not working), license re-activation or leasing is not possible until lease time is over. When lease time expires without client sync to server, license lease will be freed by the License Server. Server sync interval is defined in the license by the License Server.

Other than Floating licenses are never deactivated by Navitrol.

License validation

Successful license activation already means that license itself is valid at that time. At run-time Navitrol periodically validates the license and in case of Floating license renews the license. License validation may fail at any time for example if some license option is used without having such an option included in the license. Floating license validation may fail also if connection to License Server is lost for too long time.

There are more possibilities why validation can fail, like license expiry in case of time limited license.

Validity - time

By default licenses are created perpetual, i.e. for unlimited time. If a license has a limited Validity time, that is checked by Navitrol at run-time. Navitrol logs validity information and expiration time in text log, and optionally raises a warning WARNING_LICENSE_EXPIRATION when a time limited license is about to expire soon. This warning period is defined with a parameter license_warning_period_days. Warning can be disabled by setting this parameter to 0.

Validity - SW version

If a license has SW release version limitation, that is checked by Navitrol at start-up. Navitrol uses two digit version number (MAJOR.MINOR) to check this, which means that hotfixes are always allowed even if SW version limitation is defined.

System time

System time being wrong or being tampered with, may result in license activation or validation to fail.

Navitrol checks and monitors also system time on target to

  • be close enough the network time (Online).
  • not being tampered with, like time moved backwards.

Time is allowed to move backwards only a limited amount to allow time changes related to day-time saving. If substantially larger time adjustments are detected, this is indicated as an error when activating or validating the license.

Trial license

Trial licenses can be created for testing purposes.

NOTE: Navitrol requires that each Trial license is time limited. If a Validity time is not defined (unlimited) in a Trial license, Navitrol raises an error ERROR_LICENSE_CONTENT.

Legacy license

Legacy licensing method is supported for now for backwards compatibility. Missing license will result in immediate ERROR_NO_LICENSE_CODE, so there is no more option to drive without a license.

Legacy licensing has some practical limitations and short-comings from user point of view, like:

  • Limited license models - supports only Offline Device-locked license.
  • Limited amount of license options/features supported. Any recently created or future license options are not supported.
  • Limited forward compatibility for licenses created for other SW version.
  • License validation time can be rather long - in the worst case validation can take minutes.
  • No trial licenses.

How to set up a Legacy license

Steps to set up a Legacy license are:

  • After Navitrol has run once successfully, a license.txt file is generated in folder /home/navitec.
  • When ordering a license, one needs to define all the license options shall be included in the license.
  • Provide the license.txt file generated in target HW for license generation.
  • License is generated by the License Server and license is provided as a licensekey.txt file.
  • File licensekey.txt is copied to Navitrol run folder (/home/navitec).
  • License is validated each time Navitrol is restarted.
