NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

Motor controllers common

Updated 11.07.2024

Overview

This document describes usage and set up of motor controllers in Navitrol. The document should be used for setting up the motor controller in Navitrol.

In this document, only common parameter and troubleshooting steps are explained.
The first setup steps are explained in Motor controller specific documentation that is available for every supported motor controller (for instance, instruction for Synapticon motor controller).

Introduction

A motor controller is a device that can coordinate in a predetermined manner the performance of an electric motor. In Navitrol, a motor controller can be used to control:

  • driving motor
  • steering motor
  • actuator motor

The required number and type of motors depends on the kinematic type of the AGV and installed actuators.
In case of diff kinematics, Navitrol expects to have two driving motors. Typically, those motors should be identical and installed symmetrically.
In case of steer kinematics, Navitrol expects to have one driving motor and one steering motor.
In case of mecanum kinematics, Navitrol expects to have four driving motors. Typically, those motors should be identical and installed symmetrically.
In case of omni kinematics, Navitrol expects to have typically at least two driving motors and at least two steering motors. The number of steering wheels is set in Navitrol parameters. It is supported to have some wheels that would perform only steering without driving.
Please note that steering motor typically should have also an absolute encoder that connects to the same bus as the motor controller.

The required number of motor controllers depends on amount of motors and the number of channels in the selected motor controllers.
Typically, motor controllers are single-channel. That means that only one motor can be connected to each motor controller. Most of the supported motor controllers are single-channel, for instance, Schwarzmueller, Engel, MiControl, Oriental, Synapticon. Check List of supported devices to see all the supported motor controllers. Some motor controllers have two channels, which makes it possible to control two motors with one motor controller. For instance, Curtis and Roboteq have two channeled models.
Few motor controllers, such as some Curtins models, have three channels. Those motor controllers can be used, for instance, as all-in-one solution for automating steer or diff AGV with actuator using just one motor controller.

Requirements

For setting up the motor controller in Navitrol, it is required that:

  • Motor controller - motor connection established
    • All electrical connections are done
    • Motor parameters have been set to the motor controllers
    • See Motor controller specific documentation for connection setup instructions (for instance, instruction for Synapticon motor controller)
  • Interface bus (CAN or EtherCAT) has been wired and connected to the motor controller
  • Basic interface parameters (for CAN or EtherCAT) are set correctly. Instructions for wiring and basic parameters are described in CAN communication or Ethercat communication document.

Limitations

Functionality is limited to motor controllers that are controlled by Navitrol. Only specific motor controller and AGV kinematic combinations are supported. See, List of supported devices.

Navitrol motor control is not supported for positioning only systems

Setup

The first setup steps are explained in Motor controller specific documentation. In this document, only common parameters are explained.

Motor controller automatic disabling

In order to reduce energy consumption, Navitrol can disable drives when AGV is not moving. This involves currently all motor controllers directly controlled by Navitrol. By default, this happens 1 second after AGV has stopped moving and brakes have been activated. This feature can be configured by parameter mot_ctrl_disable_delay_ms, see table below for details.

NameTypeDefault valueDescription
mot_ctrl_disable_delay_msInt1000Motor controller disabling delay [ms] after activating brakes: -1 = never disable, 0 = no delay

NOTE: This feature is supported currently with the following motor controllers: Schwarzmueller, Curtis, Oriental, Kinco, MiControl, Synapticon and Roboteq (closed loop).

How to use

After successful setup, it is possible to drive and steer the AGV using the motor controllers (for instance, in manual mode using keyboard control).

After successful, AGV should be calibrated according to Initial Calibration Guide.

Motor controller test form

For most of the motor controllers, it is possible to manually set the outputs to the motor controllers.
This feature is useful during calibration, for instance, to tune the steering PID controller or PID for driving controllers. The benefit is that step response can be sent from this form.

This form is available for all the motor controllers except the following:

  • Roboteq
  • Nanotec
  • Syntron

The form can be opened from the "Navitrol" menu located in the top bar of Navitrol Monitor. The form is shown in figure below.

Motor controller test form

A command to be sent to a motor controller is prepared by following steps:

  • Select motor controller type - steer/drive/actuator
  • Type the motor controller's Node ID. Note: There is a special feature that is available for most motor controllers. When 255 is entered in Node ID field, all motor controllers of the selected type will receive the command.
    Note: Command is sent to the motor controller only when the correct combination of motor controller type and Node ID is selected.
  • Select output commands. Note that for every motor controller, different output commands are available and thus the form might look different.
  • Angle - available only for steering motor.
  • RPM - available for every motor. Note that when RPM is applied, the motor will start moving.
  • For some motor controllers, it is also possible to send proportional output and torque.

The prepared command is sent when "Send values" button is pressed. It can be stopped either with pressing "Stop sending" or closing the form.

Motor controller test form - Differential unit

When calibrating kinematics with differential units (Steer or Omni drive with differential unit), the user needs to verify that each differential unit is configured correctly.

The Differential unit's can be each separately tested that the unit turns to correct direction and that the speed inversions of the unit's are correct. The selected unit is visualized with a blue circle and the node ID:s of the selected unit are shown as well. After the user has selected a differential unit from the drop down menu, the desired angle or speed reference can be given the same way as for a single motor controller.

Troubleshooting

Errors and warnings

402 ERROR_MOTOR_CONTROL_FAULT

Description: Motor controller indicates fault

Error reason: Error comes from the motor controller.

How to recover: Check the error from the motor controller (for instance, using Linux terminal or from Navitrol log). Error code and error description should be printed there.

809 ERROR_AGV_CONTROLLER

Description: AGV controller alarm

Error reason: AGV controller reports error to Navitrol

How to recover: Check the error from the motor controller (for instance, using Linux terminal or from Navitrol log). Error code and error description should be printed there. Navitrol tries to reset error automatically when the error is gone.

Advanced troubleshooting

Following troubleshooting instructions are suitable for Engel, Oriental, MiControl, Curtis, Synapticon motor controllers.

Advanced troubleshooting parameter:

NameTypeDefault valueDescription
log_motor_controllersInt0Log motor controller details. 2 - Log all details

In case if there are no CAN/EtherCAT errors present, but machine is not driving, try following: Check if motor controller error is set (ERROR_MOTOR_CONTROL_FAULT and ERROR_AGV_CONTROLLER). In case if error is set, check from the Linux terminal or log file which error is set and on which controller. The keyword for search is: "FindAndLogErrors".
