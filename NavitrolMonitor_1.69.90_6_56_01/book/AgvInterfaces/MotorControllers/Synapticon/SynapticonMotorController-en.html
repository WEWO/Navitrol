Introduction

Synapticon motor controller is one of the motor controllers supported by Navitrol. Synapticon offers single channel motor controllers that can be used for different purposes (steering, driving, actuator) in the AGV.

In this documentation, instructions are provided for configuring the motor controllers so that motor controllers are compatible with Navitrol. Operation of the device is possible via EtherCAT fieldbus.

Following motor controllers has been tested by Navitec Systems:

  • SOMANET Node. This motor controller is single-channel and allows connecting BLDC motor with encoder and brakes.

The initial parameterisation/configuration of the device takes place with the software OBLAC Drives.

Currently, Synapticon motor controllers can be supported for several kinematics:

  • Diff drive (both left and right wheel)

  • Mecanum drive

Ethercat absolute encoder support is required for other kinematic models.

Commissioning steps

The following steps should be done in process of commissioning:

  1. Installation of OBLAC Drives
  2. Setting up the controller in the OBLAC Drives
  3. Installation of firmware
  4. PID tuning of the motor controller
  5. Changing Navitrol parameters

During the commissioning in OBLAC Drives, it is recommended to use the manufacturer's instruction

Commissioning should be done when AGV is off the ground.

Installation OBLAC Drives

There are two ways to run SOMANET OBLAC Drives:

  • OBLAC Drives - physical machine that comes preinstalled with Linux OS and runs OBLAC Drives

  • OBLAC Drives Virtual Machine that comes preinstalled with OBLAC Drives. It can be run on Windows, Linux and macOS.

Downloading and setting up OBLAC Drives Virtual Machine can be done using the link

Installation of firmware

  • Open Virtual Machine and wait until it is launched.

  • Type in browser the IP address shown in the virtual machine.

  • OBLAC Drives will scan the connected devices and find drives. If the drive does not have a firmware (it will be shown after connection) or firmware version is less than 5.0, new firmware needs to be installed. If new firmware is installed, this step can be skipped.

Setting up the controller

Process of setting up the motor controller is explained in details in the instruction

  • Open configuration page for the controller. Select "Launch setup wizard"

    Figure 9. Configuration - launch setup wizard

  • It will first ask you to give a name to your drive.

    Figure 10. Product name

  • Configure brakes. Enter pull voltage, hold voltage, pull time and disable delay. Note that if brakes are used, clutch-style brake control should be used since Navitrol is not commanding the output voltage. Test brakes so that they can be released.

    If there are issues with releasing brakes, check if "Safe State" active on the top of the page. If "Safe state" is active, check if safety system is powered or bypass is connected. Brakes cannot be released until safety system is powered (or safety bypass is connected).

    Figure 9. Brake set up

  • Enter the details of the motor. Pay attention to the engineering units since motor data can be scaled differently.

    Figure 11. Motor set up

  • Set up the encoder. First, choose the port where it is attached. After that, enter the parameters of the encoder. Rotate the shaft and check that it corresponds to the position of the shaft.

    Figure 12. Encoder port

    Figure 13. Encoder set up

  • Launch offset detection. It should start and finish automatically.

    Figure 14. Offset detection

PID tuning

Process of tuning the motor controller is explained in details in the configuration software.

  • Go to "Tuning" page and run system identification:

    Figure 15. System identification

  • Then go to velocity tuning.

    According to the manufacturer, it is recommended to:

    • Set all constants kp, ki and kd are set to 0.

    • Set the reference velocity to 50% of rated motor velocity [rpm].

    • Increase kP until the measured speed is around 70% of the reference speed. As a suggestion start with kp equal to 0.1, and in each step increase kp with a factor of 2

    • Increase ki incrementally to increase the speed of velocity controller (as a suggestion start with ki equal to 0.01 and in each step increase it by a factor of 2).

    • Fine tune the controller

    • The parameter kd of the velocity controller can be kept at its default value (0).

    Figure 16. System identification

Changing Navitrol parameters

  • First, set slave address to the corresponding parameter according to Table "EtherCAT address parameters for motor controllers" in Motor controllers common document. Slave addresses are given by master in direction of physical position of the devices in the bus. E.g. first device connected to master would have slave address 1 and the one connected after it would have address 2 and so on. Note that slave addresses of devices on the bus must be unique.

  • Set the communication parameters (ethercat_enabled, bus type ethercat_mode and interface ethercat_inte_primary, ethercat_inte_secondary) according to EtherCAT communication document

NOTE: Navitrol configures automatically communication related parameters in motor controller. Other motor controller settings are not set nor changed by Navitrol.

Troubleshooting

EtherCAT bus errors

  • Information about EtherCAT related errors and troubleshooting can be found from Troubleshooting section of Ethercat communication document

Motor controller issues

If motor controller indicates some errors and this error is not reset by Navitrol:

  • Try to establish connection with OBLAC Drives to check errors directly from the motor controller. Check if "Safe State" active on the controller. If "Safe state" is active, check the safety system.

  • Check Navitrol parameter I,mot_ctrl_reset_errors_timer. It should be positive in order for Navitrol to reset the error.
