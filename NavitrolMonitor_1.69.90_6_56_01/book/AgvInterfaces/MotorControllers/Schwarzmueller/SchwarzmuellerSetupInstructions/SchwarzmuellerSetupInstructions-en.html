Schwarzmueller Setup Instructions

Updated 26.01.2024

Basic Setup Instructions

  • Download service tool from powersolutions.danfoss.com and install the tool and drivers.

  • Connect the CG150 CAN-adapter to your computer and to the motor controller. Keep only one motor controller connected to the CAN at the same time during configuration.

  • Start the PLUS+1 service tool.

  • From the popup window choose the correct CAN-adapter and right channel.

  • See document Start up User Manual PLUS+1 Inverter Family V2.0 chapter 12 for more information.

  • Download the software File->File Download... e.g. 20014_STANDARD_CANOPEN_V100.lhx (see chapter 12.8.2 in manual).

    • "The selected file uses a different OS" warning can be ignored

  • Load the Service application File->Scan for Service Application File... e.g. 20014_Standard_CANopen_V100_01.p1d

    • "This Service Application file is not designed for the connected system" warning can be ignored

  • Service application let's user to modify the parameters of the device (see chapter 12.8.3 in manual).

  • Acquire parameter file (xml) for the motor from Navitec

    • Node 31 is for drive or right motor

    • Node 30 is for steer or left motor

  • The following are instructions for Navitec to generate parameter files for motors:

    • Acquire csv file for motor specific parameters (provided by Schwarzmueller)

    • Open SchwarzmuellerParamGen

    • Drag & drop the csv file into the program

    • Select controller from top left drop down menu

    • Enter ECU id of the controller as shown in System Navigator on the left, e.g. 255 as in:

  • Enter PI controller parameters (Kp, Ki, PartLoadReduc) as instructed by Schwarzmueller

  • Enter Acel & Dcel limits, values are rpm/s, usually thousands and may need to be adjusted afterwards

  • Enter maximum rpm as calculated from maximum wheel velocity & gear ratio

  • Enter encoder current range parameters (EE_MD_EncCurMax & Min)

    • If the encoders do not draw their power directly from the Schwarzmueller (5V), set the minimum current to 0,

    • Otherwise set the range according to encoder specifications

  • Save

  • In PLUS+1, select "All Parameters" (bottom of the left tree view)

Figure 1: Select "All Parameters"

  • Import parameters (xml) for motor controller (Figure 1).

Figure 2: Parameter import.

  • If loading parameters fails because of wrong ECU id, rename all "0,31" values ("0,30" for steer/left motor) in the parameter xml to the ECU id shown in PLUS+1 in the ECU column in the Parameter List and import the parameters file again

  • Select Parameter -> Check all parameters for download

  • Download parameters to Device Parameter->Download (from laptop to Motor Controller).

  • Restart device (full power cycle) to enable parameter changes (do not change view in PLUS+1).

  • Repeat for the other motor controller by disconnecting the newly configured controller and connecting the other controller.

    • It is also possible to have both controllers connected at the same time and switch the active controller but by having only one connected at the time is safer

      • Each page in the "System Navigator" view can have different controller selected so use extra caution when switching between controllers

      • Controller of each page is changed by right clicking page header -> Exchange existing ECU -> click yes to switch

      • Hovering over log variables will show the ECU id that the variable is read from, again 0,31 for drive/right motor and 0,30 for steer/left motor

Wiring Instructions

  • Connect pin 31(C1p31) to power + to enable CAN communication (Figure 3).

  • Connect pin 30 (C1p30) to power + to disable SafeStop (when SafeStop is ON connect to ground).

  • See chapter 12.6.1.1 for information about main contractor safety.

Figure 3: Diagnostics window.

  • Temperature sensor has to be connected to pins 34/35.

Basic diagnostics with PLUS+1

  • To view feedback from Schwarzmueller in real time, select any log page in "System Navigator" and press the green play button on the toolbar to start real time display

Safety Circuit

  • Click "Automated Guided Vehicle" in "System Navigator" to see an overview of the state of the Schwarzmueller

  • When Safety Circuit is broken, SafeStop should be open (pin 30 connected to ground) and show in red in the overview

  • When Safety Circuit is ok, SafeStop should be closed (pin 30 connected to +power) and show in green in the overview

  • CAN enable should also show in green (pin 31 connected to +power)

Communication with Navitrol

  • To be able to clear Schwarzmueller errors, Navitrol needs to be communicating with the motor controller.

  • To check in realtime what's happening in Navitrol CAN interface, connect to Navitrol PC with PuTTY (username: root, pw: navitrol) and type "./Navitrol dumps 0 can_inte". To stop echoing the log, press Ctrl+C

  • If Navitrol reports CAN write errors:

    • Check wires are connected, polarity is correct and both ends have 120 ohm resistors

    • Connect directly to Navitrol PC with CanKing with nothing else connected to the bus and check that data is flowing

    • Check baud rate is correct (250000) and node IDs are correct via PLUS+1 tool, if not, the motor controller parameters are incorrect update parameters and be sure to follow the instructions in Chapter 1 precisely

  • If Navitrol reports "CAN_ Obj xxx NotRxed" errors, motor controller parameters are incorrect update parameters and be sure to follow the instructions in Chapter 1 precisely

    • 21E, 19E and 29E are from node 30, it is the steer / left motor

    • 21F, 19F and 29F are from node 31, it is the drive / left motor

  • If Navitrol reports "safeStopActive" error / ESTOP, it means that the safety circuit connected to Schwarzmueller is broken (release estop / check wiring if already released).
