NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

MiControl CAN motor controller

Updated 26.01.2024

Overview

Introduction

MiControl motor controller is one of the motor controllers supported by Navitrol. MiControl offers single channel motor controllers that can be used for different purposes (steering, driving, actuator) in the AGV.

In this documentation, instructions are provided for configuring the motor controllers so that motor controllers are compatible with Navitrol.

Operation of the device is possible via CANopen fieldbus.

The initial parameterization/configuration of the device takes place with the software mcTools.

Requirements

Hardware:

  • Supported motor controller (List of supported devices document should be checked)
  • Connectors for wiring. Without the connectors, it is impossible to wire the motor controller. Types and number of connectors can be found from the mcmanual https://www.micontrol.de/en/download/documentations/mcmanual.
  • Adapter for connecting the motor controller to PC - miCAN-Stick2 https://www.micontrol.de/en/products/devices/mican-stick2](#https://www.micontrol.de/en/products/devices/mican-stick2) .

Software

  • The feature is supported by Navitrol starting from version 6.44.00.
  • Parameterization/configuration software - mcTools. Can be downloaded from the manufacturer's website: https://www.micontrol.de/en/download/software. Note that it is needed to register on a website in order to download software or device manuals.

Documentation

  • The documentation can be downloaded from the manufacturer's website: https://www.micontrol.de/en/download/documentations/mcmanual.

Limitations

  • Motor controller might be not supporting all functions (steering, driving, actuating) and kinematics. List of supported devices document should be checked to determine whether the combination of the function and kinematics is supported or not.
  • Currently, it is impossible to use MiControl motor controller with other motor controllers in the same AGV
  • MiControl motor controller support is available to position control systems only

Dependencies

This feature is dependent on CAN interface. Therefore, it is recommended to check CAN communication document.

Electrical connections

Electrical connections should be done according to the Drive Controller section in mcmanual. Connections can be divided into several groups:

  1. Power connection. Connect power to both main power supply and electronic power supply.
  2. Motor connection. Connect the motor to the motor terminals of the motor controller.
  3. Encoder connection. Connect the encoder to the encoder port.
  4. Safety connection. If the device has STO function, wire the STO-A and STO-B pins to the safety controller. Those pins should be powered whenever it is safe to drive the motor. Check Safety Manual section in the documentation for a specific controller for checking safety procedures and commissioning of the safety feature.
  5. CAN bus connection. Connect CAN wires to the CAN terminal of the motor controller. Additionally, connect the miCAN-stick2 to the CAN bus for commissioning. Wiring of the CAN bus should be done according to CAN communication document.

Setup

The following steps should be done in process of commissioning:

  • Changing the motor controller parameters
  • Changing Navitrol parameters

Changing the motor controller parameters

In order to change the motor controller parameters, connect the miCAN-Stick2 to the PC with mcToolssoftware installed. Additionally, it is recommended to read mcTools section from the mcmanual.

In order to change parameters, open mcTools. Select "Scan baudrate" and press "Scan device". The connected devices should appear to the list as shown on the Figure bellow.

   

   > Device scanning

Changing communication parameters

Once the device is connected, it is recommended to change the node id of the device. In the Tools panel, select "CAN config". In the opened window, type the Node id and baud rate. Node Id and Baud Rate should match Navitrol parameters in the subsequent commissioning steps. Node Id of each device should be unique on the bus. Baud rate should be the same for all of the devices on the bus, recommended value is 250 kBit. Save entered values to the motor controller by pressing "Set".

   

   > Changing node id

Changing motor parameters, Tuning PID controller

Those steps should be performed in accordance to the Documentation. Navitec Systems do not have successful experience in tuning motor controller. In case of issues, MiControl/supplier support should be contacted.

Note: It is possible to save the configuration of the device by opening DSA config and selecting from File menu in the left top corner the option "Save as". After saving, those parameters can be loaded from the same File menu to another motor controller.

Changing Navitrol parameters

  • Prior to communication, set the parameters (can_ch and can_bitrate_k) according to CAN communication document.
  • Since the device is supporting heartbeat, it is highly recommended to set parameter can_heartbeat_producer_enable to 1 and navigation_software_node_id to any node Id from 1 to 127 which is not used by other devices in the bus
  • Set Node ID to the corresponding parameter according to Table "CAN Node Id parameters for motor controllers" in Motor controllers common document.

Fine-tuning

  • If needed, the warning for E-STOP can be disabled from the motor controller using the mot_ctrl_micontrol_estop_check_enabled parameter
  • Automatic error reset can be disabled using mot_ctrl_reset_errors_timer.

Optional parameters

How to use

After successful setup, the motor should be rotating after giving the commands from Navitrol.

Troubleshooting

Most common issues

Errors and warnings

CAN errors

The information about CAN-bus related errors and troubleshooting can be found from CAN communication document

Motor controller errors (ERROR_AGV_CONTROLLER)

If motor controller indicates some error, Navitrol will automatically try to reset this error (unless mot_ctrl_reset_errors_timer set to zero).

If error is not reset by Navitrol:

  • Check the motor controller error code and description from Linux terminal. Try to resolve it. Power cycle the controller.
  • Try to connect using mcTools software to the motor controller. Connect it the same way as when configuring (according to the "Changing the motor controller parameters" section). Open "DSA Status" and check the motor controller parameters.
  • Contact Navitec or Motor controller support.

Advanced troubleshooting

For checking the logs, the keywords "MiControl" and "MotorController" can be used to find the information related to the motor controller operation.
