Oriental CAN Motor Controller

Updated 26.06.2024

Introduction

Oriental motor controller is one of the motor controllers supported by Navitrol. Oriental offers single channel motor controllers that can be used for different purposes (steering, driving, actuator) in the AGV.

In this documentation, instructions are provided for configuring the motor controllers so that motor controllers would work with Navitrol. Operation of the device is possible via CANopen fieldbus.

Following motor controllers has been tested by Navitec Systems:

  • BLV-R Series. It contains of two parts: driver (controls the operation of the motor) and BLDC motor with encoder.
  • Driver has 1 DC motor connection (can be used for driving/steering).
  • Only Oriental motors are compatible with the driver.

The initial parameterization/configuration of the device takes place with the support software. Initially, configuration was done by BLTS-01 software. However, currently it is recommended to use updated software - MXE-02.

Currently, Oriental motor controllers can be supported for several kinematics:

  • Diff drive

  • Steer drive

  • Steer drive with a differential sub unit

  • Omni drive

  • Omni drive with a differential sub unit

  • Mecanum drive 

Electrical connections

Electrical connections can be divided into four groups:

  1. Connecting main power supply Main power supply should be wired to the connector CN1. Special cable connector should be used.

  2. Connecting motor Motor should be wired to the connector CN2.

  3. Connecting encoder/brakes Brakes and encoder should be connected to the connector CN3.

  4. Connecting the I/O signals (CN4) Wires should be connected to the push terminals. The following pins are needed:

    a. CAN connection -- Pin 5 (CAN_L), Pin 6 (CAN_H) and Pin 7(CAN_GND)

    b. Power supply for communication interface -- PIN 20 (NET-VIN) and Pin 21 (NET-GND)

Commissioning steps

The following steps should be done in process of commissioning:

  1. Connection to support software to set up CAN communication
  2. Changing Navitrol parameters

Note: Motor controller tuning (PID tuning) is not needed with the motor controller. If needed, motor controller speed and sensitivity can be selected from some of pre-defined values in the software.

Connection to the software to set up CAN communication

Connection to the support software is done using the micro USB cable. Connect micro USB cable as it shown in the Figure 1.

It is recommended to use updated support software - MXE-02. However, for some cases, old support software BLTS-01 can be used.

Setting up CAN communication using MXE-02

Start the software. Note that it is recommended to use the software starting from version 4 in order to avoid incompatibility with older software versions of the driver.

Figure 2. Main interface of the MXE-02 software

Once window opened, click on "Select a product". The following panel should be opened in the left side of the screen.

Press "Detect communication port" (marked as step 1 on the figure) to refresh the list of devices. Once list is refreshed, select one of the options marked as step 2 on the figure.

Figure 3. Connection to the driver in MXE-02

Once connected, press "Start the simple settings". Software should establish connection to the motor controller.

After that, select desired NodeId and Baud Rate of the motor controller. NodeId and Baud Rate should match Navitrol parameters and NodeId of each device should be unique on the bus. Save those values to the motor controller by pressing "Reflecting on the driver".

Figure 4. Configuration in MXE-02

Now, motor controller is ready for communication with Navitrol.

Setting up CAN communication using BLTS-01

Start the software.

Figure 5. Main interface of the BLTS-01 software

Once window opened, click on "Setting of communication". Select ORIENTAL MOTOR COM port.

Figure 6. Connection to the driver in BLTS-01 software

Press "Ok" and window should be closed.

Next, press "Data Reading (Product -> PC)". The following window should be opened:

Figure 7. Configuration in BLTS-01 software.

Press "Start the simple settings". Software should establish connection to the motor controller.

After that, select desired NodeId and Baud Rate of the motor controller. NodeId and Baud Rate should match Navitrol parameters and NodeId of each device should be unique on the bus. Save those values to the motor controller by pressing "Reflecting on the driver".

Now, motor controller is ready for communication with Navitrol.

Changing Navitrol parameters

  • First, set Node ID to the corresponding parameter according to Table "CAN Node Id parameters for motor controllers" in Motor controllers common document.

  • Set the communication parameters (can_ch and can_bitrate_k) according to CAN communication document. In addition, set can_sync parameter to value 1.

  • In order to enable heartbeat, set parameter can_heartbeat_producer_enable to 1 and navigation_software_node_id to any nodeId between [1..127] which is not used by other devices in the bus.

  • It is highly recommended to enable heartbeat for safety reasons.

  • Other motor controller settings are done using SDOs by Navitrol.

Troubleshooting

CAN-bus errors

  • Information about CAN-bus related errors and troubleshooting can be found from Can Communication document

Motor controller issues

If motor controller indicates some errors and this error is not reset by Navitrol:

  • Try to establish serial connection to check errors directly from the motor controller. To check errors, connect with BLTS-01 or MXE-02 software. Connect it the same way as when configuring (according to the instruction above). Press "Start the simple settings". In the left bottom menu choose "Alarm monitor".

  • Check Navitrol parameter I,mot_ctrl_reset_errors_timer. It should be positive in order for Navitrol to reset the error.

  • Some errors cannot be reset from the CAN bus by Navitrol (for instance, overvoltage or encoder disconnection). In such a case, a power cycle is required for the motor controller.
