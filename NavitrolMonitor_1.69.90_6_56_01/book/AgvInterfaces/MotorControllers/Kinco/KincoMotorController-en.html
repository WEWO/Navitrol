Overview

Introduction

Kinco offers single channel motor controllers that can be used for driving in the AGV.

This document provides instructions for configuring the motor controllers to be compatible with Navitrol.

Requirements

Hardware:

  • Supported Kinco Motor controller models can be found from the List of supported devices.
  • Connectors for wiring (or alternatively Kinco cables with connectors). Without the connectors, it is not possible to wire the motor controller. Types and number of connectors can be found from the Technical Manual for the motor controller.
  • Adapter for connecting the motor controller to PC. RS232 cable is required by the KincoServo+ software.

Software

  • Motor controller related software can be downloaded from: Kinco Download site.
  • The following item should be downloaded for the commissioning:
    • KincoServo+ for parameters adjustment and diagnostics

Documentation

Technical manual is needed for the motor controller. It contains dimensions, wiring instructions and CANOpen messages. Additionally, it contains the commissioning instructions and the SDO object dictionary. Technical manual depends on the motor controller model and can be found on the Kinco Download site.

  • The manual for OD1X4S and FD1X4S models:
    • Kinco Low-Voltage User Manual
  • The Catalog contains some technical information of the driver and motor.
    • Kinco Catalog Low voltage servo

Limitations

  • Motor controller might not support all functions (steering, driving, actuating). See List of supported devices for information about the supported functionalities.
  • It is not possible to use Kinco motor controller with other motor controllers in the same AGV

Dependencies

  • CAN communication.

Electrical connections

The connections might vary depending on the motor controller model. See Kinco Download site for further info.

Quick setup

The following steps should be done during commissioning:

Step 1 (Option 1): KincoServo+

  • Change Node-ID and baud rate with KincoServo+

Step 1 (Option 2): General CAN-analyzer Commissioning

Step 2

After the Node and baud-rate are set on all motor controllers, proceed to:

  • Navitrol parameterization and controller tuning

KincoServo+ Easy Commissioning

Kinco offers KincoServo+ for easy commissioning. It can be downloaded from Kinco Download site.

Change Node-ID and baud rate

  • Download KincoServo+ software and connect your PC to the motor controller (communication interface RS232).

  • The default baud rate for RS232 is 38400 and COM ID 1. If user does not know the COM ID, broadcast address 127 can be used instead.

  • The default CAN Node-ID is 1. It can be changed from Driver -> Driver Configuration -> Node_ID.

    Change the Node-ID

  • Change the CAN baud rate. Default baud rate for Kinco motor controller is 500 kbd. Please note that the Kinco motor controller and Navitrol baud rate needs to match.

    Change the baud rate.

  • Go to the Driver -> Init Save Reboot and click Save Motor Parameters, after that click Reboot.

    Save Motor parameters

  • Repeat the same procedure for all motor controllers.

General CAN-analyzer commissioning (Option 2)

If you do not have the possibility to use KincoServo+ software, the commissioning needs to be done by using CAN analyzer. With this method each motor controller needs to be connected and configured separately on the bus.

Controller switch configuration

  • Connect the motor controller to your PC with a CAN-USB adapter.

  • NOTE: The controller might have DIP switch that activates 120-ohm resistor. A CAN-bus needs 120-ohm resistors on both ends of the bus

Navitrol parameterization and controller tuning

Node-ID and CAN bus bitrate

Node-ID defines the address of the CANopen device and it must be in range from 1 to 127. In addition, the Node-ID has to be unique for each device on the CAN bus.

CAN bus bitrate (baud rate) is the communication speed. By following these instructions, this speed is set to 250 Kbit/s for the motor controllers. The speed needs to be the same for Navitrol and motor controllers, otherwise the CAN bus communication will not work.

Every motor controller has a Node-ID. params.txt file has a parameter for each motor controller Node-ID and a parameter for Navitrol CAN bus bitrate.

First, set Node-ID to the corresponding parameters:

  • mot_ctrl_drive_can_nodeid_arr

Set the communication parameters (can_ch and can_bitrate_k) according to CAN communication document.

PI-controller values tuning

All Kinco motor controllers have an internal PI-controller that must be tuned. These controllers must be tuned. Rule of thumb instructions to find the optimal parameters are presented below (reference from Kinco CANopen document).

*1. According to oscillation and noise sensed with the finger and the ears: This method is based on experience,but it’s efficient. The user can listen to or touch the machine, at the same time increasing and reducing the kvp. When an acceptable maximum kvp value is found, the current setting can be specified as the maximum velocity loop bandwidth.

  1. According to the scope image from KincoServo+: The user can create a jump command for velocity control and sample actual velocity and current while changing kvp. The right velocity curve should quickly fulfil the command without oscillation and unusual noise.*

Motor controller can be tuned

  • Modifying the Kvp and Kvi/32 parameters with KincoServo+ software. After the optimal parameters are found, save those to the Navitrol parameters (params.txt)
  • Modifying Navitrol P- and I-gain parameters with Navitrol and Navitrol Monitor. When Navitrol starts, it will always override the motor controller P- and I-gain values based on Navitrol parameters.

The P-gain can be set with mot_ctrl_drive_p_arr. These values are set to Kvp[0] (SDO Object Index 0x60F9 SubIndex 0x01) parameter from Kinco documentation. The I-gain can be set with mot_ctrl_drive_i_arr. These values are set to Kvi/32 (SDO Object Index 0x60F9 SubIndex 0x07) parameter from Kinco documentation. The Kvi[0] (SDO Object Index 0x60F9 SubIndex 0x02) is not used with Navitrol, it is kept as 0.

Troubleshooting

Motor is not spinning when sending RPM value to it and there is no error code

The encoder resolution might be incorrect. Kinco communication protocol is not using RPM value for speed, but instead Kinco internal decimal value that Navitrol will calculate from RPM based on encoder resolution parameter. By default, these parameters are 0 and with that value the measured RPM and target RPM are so low that the motor is not spinning.

Citation from Kinco document:
For example, engineering unit of speed is rpm and internal unit is dec. The relationship is: 1RPM = 2730dec (Encoder resolution 10000)! It is assumed that if the speed is 10rpm, user need to write 27300dec (6AA4 in HEX) when use communication to control. The decimal value is calculated from this formula: DEC = [(RPM * 512 * encoder resolution) / 1875]

Check that these parameters have correct values.

  • mot_ctrl_drive_encoder_pulses_per_rev and mot_ctrl_drive_encoder_pulses_per_rev_arr

CAN-bus errors

If CAN-bus is not working correctly, make sure that following things are checked:

  • CAN-bus has 120 ohm resistors activated on the both end of the bus. The total CAN-bus impedance has to be approximately 60 ohms.
  • If the motor controller has PID switch for the resistor, check that it is set to correct position.
  • Check that all motor controllers and other CAN-devices in the bus have same baud rate
  • Make sure that all CAN-devices on the bus have individual Node-ID
  • Double-check Navitrol parameters in this document
  • Make sure that you have also done CAN-bus grounding
    • If CAN-bus GND is not grounded, there might occur error frames in the CAN-bus
    • Higher baud rates require CAN-bus grounding because they require more accurate voltage levels
  • More information about CAN-bus related errors can be found from Navitrol CANOpen Communication Troubleshooting.
