NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

Optical Code Reader

Updated 26.01.2024

Glossary

TermDescription
TCPTransmission Control Protocol - connection oriented network communication protocol
SOPASSICK SOPAS engineering tool is a device configuration SW provided by SICK AG
Read GateRead Gate control is the term used for SICK Lector reading activation

Supported models

For now Navitrol supports SICK Lector 6xxx using Ethernet interface (TCP). SICK Lector is an image-based code reader capable to read many different type of QR-code and bar-codes. It also has limited capability to tell how far from centerline the reader is.

Device configuration

SICK Lector 6xxx

Prior to configuration:

  • SICK SOPAS software should be downloaded and installed.
  • IP address of the connected PC should be changed to match the domain with that of SICK Lector device.
  • SICK Lector shall be connected to PC using Ethernet and powered up.

SICK Lector shall be configured both for intended application and for communication with Navitrol. There are two optional methods to do this configuration.

  • Configure each SICK Lector with SOPAS, one by one
  • Configure SICK Lector by Navitrol using file exported from SOPAS

In both options at least one SICK Lector needs to be configured using SOPAS. When configuration has been done, it can be stored to a file in different formats; to be used by SOPAS for later use or in another format to be used by Navitrol.

The following shows how to connect to the device using SOPAS and settings needed for the device to communicate with Navitrol in expected manner.

Connect to SICK Lector 6xxx with SOPAS

Connection starts by searching the devices. First click A. "Search devices". Connected device(s) should then appear in the list B. Each Lector has two optional connections (ports 2111/2112).

Double click the second connection in the list and you should see the device on the left side pane.

Double click the "icon" of the device on the left side pane and select Advanced user interface.

Finally you should be connected to the device and tree like settings selections are visible on the left side pane.

Reset, read, write SICK Lector 6xxx configuration with Sopas

First decision point is whether one would like to do configuration by

  • Starting with factory defaults OR
  • Further modifying existing settings

New device is in factory defaults when delivered. If one wishes to force device to factory defaults at any time, then select from menu: "DeviceName" - Parameter - Load factory defaults in device.

As noted by SOPAS, also communication parameters will go to factory defaults. IP address shall then be set to right value as shown below.

If one would like to modify existing configuration instead, then existing configuration can be read from device by clicking the icon with green up arrow.

Then finally when configuration is done, it can be written to device by clicking the icon with red down arrow. And finally saved permanently by clicking memory chip icon.

One can also save the project in SOPAS for later use (see SOPAS documentation) or export the settings as explained in later chapter.

Configure SICK Lector 6xxx with SOPAS

Configuration for the device to communicate in expected manner with Navitrol is presented in figures below. Only those settings that are different from device default settings are shown. Primary objectives with these settings are immediate response when code (QR-code, bar code,..) is detected and continuous communication with reasonable interval for device monitoring purposes.

Object Trigger Control

Read Gate shall be controlled by Navitrol using SOPAS commands. We also want immediate "Good Read" message sent out as in some features repetitive reading is necessary.

Evaluation Conditions

Output Control

In most applications we want immediate response without any delays.

Output Format

It is vitally important to have right output format for "Good Read". Failing to comply with this format will result in Navitrol trashing the message sent by the device.

NOTE: Even though the format must be exactly as shown here, Navitrol SICK Lector interface does not require all data fields to be present. Message has to start with "GoodRead;" as shown here, but then minimum one out of the four data fields (ID, X, Y, Z) is required. Navitrol SICK Lector interface reports which fields have been read and it finally depends on the Navitrol feature used what data is truly needed. Even if the Navitrol feature in question would not need all data fields, there is no benefit really for defining different output message than that shown in figure.

Ethernet

Heartbeat message with interval of 1 sec is required by Navitrol for device monitoring purposes.

Configure SICK Lector 6xxx by Navitrol

When one device has been fully configured with SOPAS, one can export the settings and corresponding configuration commands to a file as shown below.

This results in ASCII-file with all commands that are needed for device configuration by Navitrol.

In order for Navitrol to use this configuration file, it shall be stored to same folder as Navitrol executable and params.txt for example. Then the file name is defined in Navitrol parameters. After that Navitrol will use the file at startup and after any communication break to send all those commands to the device. This feature makes it also simple to replace SICK Lector with new one if needed, as Navitrol takes care of writing the same configuration to a new device. See chapter Navitrol configuration for further details for parameters involved.

NOTE: First command in configuration file is for setting Access Mode required for configuration. Last HEX value is the HASH password for the device. If devices have different passwords, then this should be modified for each device separately.

Navitrol configuration

The SICK Lector camera is regarded as an external localization sensor and therefore the following parameters shall be correctly set. Most of the parameters are array parameters; valid indexes being 0...3 and indexing should start with 0.

Parameters for basic communication and configuration

ParameterDescription
I,ext_loc_eth_model[],Model of ethernet external localization sensor: 0 = PGV camera, 1 = reserved, 2 = SICK Lector
S,ext_loc_eth_ip[],IP address of the device
I,ext_loc_eth_port[],Communication port of the device (with SICK Lector 2112 with defined configuration)
S,ext_loc_config_file[],SICK Lector sensor configuration file name, if such exits (*)
I,ext_loc_config_msg_interval_ms[],SICK Lector sensor configuration command transmit interval [ms]. Defaults to 100 ms.

(* NOTE: Defining configuration file name means that configuration made by Navitrol is required. If defined file does not exist, Navitrol will report configuration failure and SICK Lector interface will not work.

Parameters for log control

ParameterDescription
I,log_ext_log,Log communication details, 0 - min level ... 3 - max amount of details.

Parameters for mounting - feature & application specific

ParameterDescription
I,ext_loc_x_offset[],SICK Lector longitudinal position offset from navigation point
I,ext_loc_y_offset[],Camera transversal position offset from navigation point
I,ext_loc_y_offset_deg[],Heading offset in degrees between machine coordinate system and camera coordinate system, heading angle of focal axis when vertical
I,ext_loc_focal_orientation_deg[],Orientation of camera coordinates around focal axis when the axis is horizontal. The rotation angle should be such that reported detection X is to the left and Y up when seen behind the camera to the markers

NOTE: For these parameters see further details in in feature specific documentation

WARNINGS

Navitrol SICK Lector interface does not raise any ERROR, but it can raise WARNINGS.

WarningDescription
WARNING_CODE_READER_NO_COMMSNo communication to the device. This WARNING should not be set after startup.
WARNING_CODE_READER_STATUS_NOT_OKError reported by device itself or device configuration error. Configuration error can happen for example if defined configuration file does not exist or device password is not right in configuration file.

NOTE: SICK Lector startup time is quite long - around 1 minute. So it is normal to have some WARNING after startup for longer period than with most other sensors.
