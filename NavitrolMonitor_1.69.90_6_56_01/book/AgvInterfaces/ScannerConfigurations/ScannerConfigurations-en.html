Additional measurement scanners

In addition to localization scanners (front/rear), Navitrol supports two additional measuring scanners. Measurement scanners can trigger slow down and stop based on warning zones if parameter wallmon_scanner_zone_mode is non-zero. Any scanner model supported by Navitrol can be defined to be used as measurement only scanner as well. These scanners can be used for

  • Obstacle avoidance

  • Target detection

Scanner reset

When scanner has been configured in the Sensor Studio and the configuration is not known, then it may become impossible to connect to it. In such a case it may become necessary to reset it using the following procedure.

  1. Switch off the scanner

  2. Remove scanner from the CU - mounting base (2 in the Figure)

  3. Remove Jumper PNP/NPN (1 in the Figure) like it is shown on the figure

  1. Attach scanner on the base

  2. Switch on the scanner

  3. Message "HIDE COVER  TO RESET CONFIGURATION" should be displayed on the scanner

  4. Within 2 minutes: Cover the screen (360°) with a cloth, so that all measurement values are small than 150mm.

  5. If successful , the scanner deletes the configuration; „CONFIGURATION WAS RESET -- WAITING FOR CONFIGURATION"

  6. Switch off scanner

  7. Remove scanner from CU

  8. Insert Jumper PNP/NPN

  9. Attach scanner to CU

Sick

S300

There are several variants of S300 available. S300 Expert feature list includes "Extended measured data output via RS-422 with landmark recognition", which means that

  • S300 Expert can transfer all measured data in real-time (i.e. 40 ms interval), if the interface is set to 500 kBaud and no filter is set.
  • S300 Expert has built-in reflector detection (landmark detection).

Other S300 variants do not support reflector detection and can transfer only every second measurement (i.e. 80 ms interval), if the interface is set to 500 kBaud.

So, if reflector detection is needed, then Expert model S300 scanners must be used. Because of capability to transfer all measurement data in real-time, S300 Expert is strongly recommended for localization use in all cases.

Configuration of the S300 is done with SICKs CDS-software.

Configuration as Localization Scanner

The S300 scanners should be configured as described in this section to be used as localization scanners. It should be noted that, for communication protocol version 1.03 and later, it is not possible to get warning zone information from the scanners when used for localization, only protection zone information is available.

The baud rate has to be set to 500 kBaud, the send mode needs to be Continuous data output and the Measured data output has to be Distance with no ranges selected. Also, the Message structure must be 1 message and all I/O data outputs need to be unchecked. These configurations can be done from the Measured data output-tab.

Also the field mode has to be set to 4 x 0.5 from the Resolution/scanning range-tab.

Set the following parameters to params.txt:

I,scanner_model_front/rear, 1

S,front/rear_scanner_serial_port, "The name of the serial port that the scanner is connected to

F,reflector_rssi_min_ratio, 0.75

Configuration as Safety Scanner

The S300 scanners can be configured as safety scanners with both warning zone and protection zone information available for communication protocol version 1.03 and later. The only difference in the configuration software from the localization case is that OSSDs from I/O output needs to be enabled as seen below. All the other settings should be the same as in localization scanner configuration.

In params.txt file, the following parameters should be set:

I,safety_scanner_model[0], 1

S,safety_scanner_addr[0], "The name of the serial port that the scanner is connected to"

Safety zone configurations

By default, only two zone triggers (one protection zone and warning zone) are used. To have two warning zone violations reported to Navitrol, extra configuration is needed, both in Sick Safety Designer as well as Navitrol parameters. Scanner has to be configured to use cutoff paths to communicate violated zone to Navitrol.

Following instructions for scanner configuration was provided by Sick

Configuring with Safety Designer.

After creating a project in Safety Designer with chosen nanoScan3 device and optionally accessing the device with USB, you should be able to see the measurements from nanoScan3 in the "Fields" configuration window. The configuration file can also be made without the device.

The field coordinates window can be zoomed using CTRL+Scroll

The first protective field can be drawn with the rectangle-tool circled with yellow marker. In this example protective field is 500 mm wide, 800 mm high ranging from 200 mm to 1000 mm. Software automatically adds a triangular shape in front of the protective field, so blinding the scanner would not prevent safety output from activating.

Warning fields can be added using the top right marked button "Add field to field set" and it can be set as a "Warning field" by choosing "Field type" in the bottom right corner. After that you can similarly define the field with "Draw rectangle" or with other shape tools.

Below we can see 3 fields. One protective field defined from 200 -- 1000 mm, two warning fields with 1000 - 1500 mm and 1500 -- 2000 mm ranges respectively.

After defining the fields, move to "Inputs and outputs, local" to define the digital outputs. In this example we assign two "Monitoring results" into inputs 2 and 3.

After defining the outputs, move to "Monitoring cases"-tab to assign the fields defined before to the outputs, by dragging the field set. After that tick the "Universal outputs", so the safety scanner knows which outputs to activate, when field is intercepted.

Now you have configured the scanner to use two warning fields and one protective field. Next step is to send the configuration to the scanner if you have one available by clicking "Transfer", or to save the configuration for later use.

Safe Cut-off path needs to be configured for protective field, and Non-safe Cut-off paths for warning fields. In the example case above the cut-off paths were set to following values:

  • Protective field has safe cut-off path 1

  • Warning field 1 has non-safe cut-off path 2

  • Warning field 2 has non-safe cut-off path 3

These values we need to configure to the Navitrol parameters.

Navitrol parameters

Navitrol parameters are configured to same path numbers (1-20) as defined in Safety Designer to map fields into zones in Navitrol. After fields are defined and configurations done in both Safety Designer and Navitrol, configuration should be verified by connecting Navitrol Monitor and triggering the scanner fields to see if matching warning is raised in Monitor.

Keyence

SZ-V32N/SZ-V32NC

SZ-V series is a modular design safety scanner, consisting of two main components; scanner head and display unit. Different component combinations then result in different device variants. From Navitrol point of view the relevant thing is to have display unit having network interface and device having 'N' in model name:

  • No N: No network connection capability

  • N: With PROFIsafe (PROFINET, UDP, EtherNet/IP™)

  • NC: With CIP Safety™ (EtherNet/IP™, UDP)

Connecting

The picture below shows different options for connecting SZ-V series devices to network. Complete units (having both head and display unit) can be connected directly to IPC or via switch. They can be connected "daisy-chained" using dual connectors in display unit.

SZ-V series devices also support master-slave connection, where head units without display units are connected to full unit via special expansion cable; see red rectangle in picture above In such a case communication to slave units (heads) can be realized through the master device.

NOTE: Navitrol does not support this master-slave communication method.

Operating modes

SZ-V series has two operating modes; Standard Mode and High Speed Mode.

High Speed Mode is required by Navitrol. It does with some shortcomings though.

The measurement distance data can be received in increments of 0.2° in standard mode and in increments of 0.4° if the Operating mode is High Speed mode.

Keyence also has an option for "measured value stabilization filter"; which is not recommended as it adds extra delay to measurements.

Configuration

Configuration can be done using Keyence SZ-V configuration software SZ-V Configurator; included in the Safety Device Configurator. It is possible to communicate with the SZ-V configuration software (SZ-V Configurator) via Ethernet or USB.

Detailed step by step instructions for different connection options are described in "Keyence Safety Laser Scanner SZ-V32N / SZ-V32NC Communications Manual". The following pictures show the relevant communication settings when used with Navitrol. Communication protocol shall be UDP and scanner should have a static IP address.

Pepperl+Fuchs

R2000

The only manual setting required is the IP address setting of the Ethernet connection of the scanner. The Navitrol application handles the other scanner settings at startup. There are two methods to edit the IP setting; local user interface and PACTware software.

Using PACTware software

  • Download the PACTware software and Device Type Manager (DTM) from the software section on the page: pepperl-fuchs.com

  • Connect the scanner to the computer via Ethernet cable.

  • Set your computer IP address to 192.168.0.xxx (the scanner default IP address is 192.168.0.1).

  • If the scanner is restored to factory default settings just set your computer to DHCP mode (the scanner is in Auto IP mode).

  • Start the PACTware software.

  • In the Device catalog, select communication by double clicking the R2000 IP Comm (see Fig 10). The selected IP communication should appear in the Structure window.

Figure 10: Select communication.

  • Select the device in the Device catalog by double clicking the proper device model, like OMDxxx-R2000 UHD (see Fig 11). The selected device should appear in the Structure window (see Fig 12).

Figure 11: Select device.

  • Select R2000 IP Comm in the Structure window and click connect in the top bar. Do the same for OMDxxx-R2000. If a connection is established the connection picture in the Structure window should turn green (Fig 12).

  • When connected, double click the OMDxxx-R2000 (Fig 12) to open a scanner configuration window (Fig 13).

Figure 12: Connect.

In the scanner configuration window (Fig 13):

  • Select the Ethernet Setup.

  • Tick Enable update of IP configuration.

  • Change the IP address in the New Settings.

  • Store the settings in the device.

  • Reboot the scanner or turn it off.

Figure 13: Ethernet Setup.

Using local user interface

Local user interface can be used to change the IP address settings.

Buttons 3 and 4 can be used to navigate into local menu system for acquiring and modifying some settings, like IP settings. Visual feedback is presented in area marked with 7. Local menu is pretty intuitive to use -- menu structure and more detailed instructions can be found from P&F OMDxx-R2000 user manuals.

NOTE: Changes to the IP configuration take effect only after a restart.

Navitrol parameters

Set the following parameters to params.txt:

I,scanner_model_front/rear, 3

S,front/rear_scanner_ip, "The IP address that was set for the scanner"

I,front/rear_scanner_port, 80

F,reflector_rssi_min_ratio, 0.75

There are also several P&F R2000 specific parameters in params.txt. Available and recommended settings for these values depend on R2000 device family. See chapter R2000 variants for details.

R2000 variants

Quanergy

Quanergy M1

Quanergy M1 comes in different variants; M1-CORE, M1-PLUS and M1-ULTRA (consult Navitec for variant selection). They differ in some features like maximum scan distance. Configuration for each model is however the same.

The Quanergy M1 Sensor Settings Management application (web server) runs in a web browser over the HTTP communication protocol to support a direct connection between the web server (scanner) and client. For details see "Quanergy M1 Sensor User Guide". The following lists some basic settings that shall be set for navigation use.

NOTE: Reflectors can't be used for localization with Quanergy M1 scanner.

Connecting

Connect the scanner to any PC with browser.

  • Open a web browser.

  • In the browser URL field, type the URL, which is simply the sensor IP address appended by :7780. For example 192.168.1.180:7780

  • Press the Enter key. The M1 Sensor tab appears

By default, the login credentials are:

Username: editor
Password: <sensor_serial_number>

Settings

Relevant settings can be found from "Settings-Edit Settings" and "Settings-Edit Network Settings".

Network settings

Set static IP address of your choice and "Submit" and "Reboot".

Sensor settings

Select maximum frame rate of 20 Hz. This results in roughly 0.134 degree angular resolution.

It is recommended to use Return data: Return 0. If any other data selection is used, then

  • single return -- Navitrol obviously uses the data set selected

  • multi return (ALL) -- Navitrol picks return 0 value (causes unnecessary extra network load)
