NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

Supported devices

Updated 13.09.2024

General

This document lists all devices that are compatible with Navitrol navigation software.

Support for a non-compatible device will require custom development.

CAN cards

A CAN card must be installed to the navigation IPC to connect Navitrol to a CAN Bus.

Models:

  • Kvaser
  • Peak
  • SocketCAN
    • Any hardware supporting SocketCAN

Motor controllers

Motor controller support depends on the kinematic type of the AGV. For example, if a motor controller works with an omni drive, it won't necessarily work with a differential drive. However, adding support for a new kinematic type is typically much easier than creating support for controller not listed here.

Schwarzmueller

A Navitec standard motor controller for AGV purposes. The performance has been good and reliable in multiple projects.

Suitable for big and medium sized AGVs.

Interfaces:

  • CANopen

Models:

  • MIxx

Supported kinematics:

  • Diff drive
  • Steer drive
  • Omni drive
  • Mecanum drive

Supported actuators:

  • Reference controlled hydraulic actuator
  • Reference controlled rotating actuator

Engel

An integrated motor controller. Not recommended as it does not have safety logic in case of communication failure.

Suitable for small and medium sized AGVs.

Interfaces:

  • CANopen

Models:

  • HFI series
  • Note - Other models might work but they have not been tested

Supported kinematics:

  • Drive motor for steer drive in combination of Schwarzmueller for steering

Supported actuators:

  • None

Nanotec

A reliable motor controller. Can have integrated or separate driver.

Suitable for small and medium sized AGVs.

Interfaces:

  • CANopen

Models:

  • PD2-C
  • PD4-C
  • PD6-C
  • N5-X-2
  • Note - Other models might work but they have not been tested

Supported kinematics:

  • Diff drive
  • Mecanum drive

Supported actuators:

  • None

Roboteq

A good motor controller for various range of power and motor types.

Suitable for small, medium and big sized AGVs.

Interfaces:

  • CAN
  • Serial USB/RS232 (open loop)
    • Not recommended for production use due to insufficient signal noise immunity

Models:

  • All models with a CAN or serial interface

Supported kinematics for open loop:

  • CAN dual channel controller
    • Diff drive
    • Steer drive
    • Steer drive with a differential sub unit
    • Omni drive
    • Mecanum drive
  • CAN Single channel controller
    • Diff drive
    • Steer drive
    • Mecanum drive
  • Serial dual channel controller
    • Diff Drive
    • Steer Drive
    • Mecanum Drive

Supported kinematics for closed loop:

  • Diff drive
  • Mecanum drive

Supported actuators:

  • None

Curtis

Suitable for big and medium size AGVs.

  • CANopen

Supported actuators:

Supervisor interface

An interface designed by Navitec over which Navitrol provides control references and the Supervisor (connected device e.g. PLC) provides odometry and status data. With this option, the Supervisor device is responsible for sending the references to the motor controller.

The interface makes it possible to use any motor controller that the Supervisor (e.g. PLC) can control.

Motor controllers <-> Supervisor <-> Navitrol

Interfaces:

  • Navitec supervisor protocol (Ethernet TCP/IP)

Models:

  • Any device able to implement the protocol
    • Typically a PLC

Supported kinematics:

  • Diff Drive
  • Steer Drive
  • Diff-Steer Drive
  • Omni Drive

Supported actuators:

  • Binary controlled actuator
  • Reference controlled actuator

SIMOVE

SIMOVE is Siemens modular software platform for mobile robots. Navitrol is the navigation module in the system and communicates to customer's onboard device (e.g., PLC) via the interface specified by SIMOVE. The interface is kinematic free, thus the PLC/device onboard controls the motor controllers of the AGV.

Interfaces:

  • SIMOVE protocol, Ethernet TCP/IP (UDP)

Models:

  • Any device able to implement the protocol
    • Typically a PLC

Supported kinematics:

  • Kinematic free

Supported actuators:

  • None

General IOs

General IOs can be used for various purposes. Typically they are used as communication interfaces between Navitrol and the device (PLC) controlling the AGV. For example, status data is typically shared between the devices.

Ethernet supervisor

An interface designed by Navitec to provide and get data from the AGV. Can be used for example to provide BMS data, AGV status, giving drive orders and setting speed limits.

CAN supervisor

An interface designed by Navitec to provide and get data from the AGV. Does not support as many functions as the Ethernet supervisor

Modbus Ethernet TCP

Modbus Ethernet modules can be used for IO. Navitrol supports Beckhoff EK9000 Bus Coupler but no generic interfaces exist.

Note: Non generic interface - will require custom development to use

Custom CAN interfaces

Navitrol supports custom CAN interfaces but they will need custom development.

Note: Non generic interface - will require custom development to use

Scanners

The main purpose of scanners is localization based on natural features. Other use cases include safety, target detection, obstacle detection and reflector based localization.

For localization and target detection use the recommended angular resolution is 0.2 degrees or better. Consult Navitec if considering other options.

Data matrix line cameras

Data matrix line cameras are used for localization based on QR-tapes. QR-tape based localization is more accurate compared to scanner based one. It is typically used to increase accuracy of high precision docking tasks.

Pepperl+Fuchs

Interfaces:

  • CANopen
  • TCP/IP

Models:

  • PGV100-xx
    • Should be installed at height of 100mm from the QR-tape
  • PGV150-xx
    • Should be installed at height of 150mm from the QR-tape
    • Recommended as it has larger field of view

QR code and bar code reading devices

QR code and bar code reading devices can be used for example identifying resources. Taking one into use will require specifying the use case with Navitec and some custom development.

SICK

Interfaces:

  • Ethernet

Models:

  • Lector 6xx
    • Able to detect the exact position of a resource.

Inertial measurement units (IMU)

IMUs are used in addition to scanner localization to increase localization accuracy by improving the odometry data.

LP-RESEARCH

Interfaces:

  • USB
    • LPMS-CU2
    • LPMS-CU3
    • LPMS-URS2
    • LPMS-URS3
    • LPMS-IG1
  • CANopen
    • LPMS-CU2
    • LPMS-CU3
  • Serial
    • LPMS-URS2
    • LPMS-URS3

Models:

  • LPMS-CU2
    • Sufficient performance for most localization cases
    • Not manufactured anymore
  • LPMS-CU3
    • Replacement for CU2
  • LPMS-URS2
    • Performance is similar to CU2
  • LPMS-IG1
    • More accurate and more expensive than the other models
    • The extra accuracy makes difference only when driving long distances without other localization sources

GPS devices

GPS devices are used for localization in outdoor environments that do not have enough static environment for laser scanner based localization. Note: When using GPS an IMU is also required for detecting the tilt angle of the GPS sensors.

      • Note: Not tested by Navitec but Manufacturer states that the interface is compatible

Absolute encoders

Encoders are used to measure the angle of rotating axle. These measurements are transformed into odometry data, such as driven distance and wheel angles.
