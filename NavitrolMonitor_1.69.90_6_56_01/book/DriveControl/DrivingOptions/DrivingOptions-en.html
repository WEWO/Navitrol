NAVITROL USER INSTRUCTIONS - Navitec Systems Oy CONFIDENTIAL

Driving Options

Updated 26.01.2024

Overview

The purpose of this document is to give reader an overview of the different alternatives that exist for driving the AGV. There is a vast variety of different tools and interfaces from which the vehicle can be commanded to drive. This is to ensure that there are enough options to cover the needs of different productions, and to provide effective tools for calibration and testing. To achieve smooth and accurate driving, there are also some tips on how to affect the driving performance.

Driving manually

Manual driving refers to the cases where the reference values given to the AGV are coming from a manual source rather than from a predefined trajectory. It is needed for different purposes, e.g., teaching the environment, initial calibrations and driving vehicle out of production for maintenance. Navitrol provides a few different ways to handle this.

Web interface

Web interface gives a joystick-like control for manual driving. The user can type the IP address of the ethernet port in IPC to a browser e.g., in mobile phone and open the user interface. See more information of the Web interface in its own document.

Navitrol - WebInterface Instructions

Keyboard control

Keyboard control can be used in Navitrol Monitor. It can be enabled from the Navitrol selection (See Figure 1) or by holding Shift-key whereafter the arrow keys are used to set the reference values. In some situations, this may not offer enough degree of freedom to accurately perform a certain driving sequence. Thus, keyboard control is usually used only for initial calibration and testing. For more information, see the document Navitrol Monitor - User Instructions.

Figure 1: Keyboard control

Ethernet Interface

The ethernet interface gives customers the freedom to use their owndevice to control the vehicle manually through Navitrol. There are two Supervisor status message sections that are offered: Remote control section and Manual section. The definition for these can be found in the Ethernet Communication specification between Navitrol and AGV supervisor document.

The remote-control section allows user to have the same joystick-like control as the previously mentioned Web interface and relies on the same parameters.

Manual section gives a keyboard-like control, so driving forward, backward, turning left and turning right. The speed can be set in the message itself while the turning rate is defined using parameters:

F,veh_inte_manual_ang_vel 0.785 # turning rate in radians per second F,max_turn_rate_deg_p_s 30.00 # maximum limit for turning rate (deg/s)

Driving automatically

In automatic driving mode, the vehicle follows a virtual route to a target location and a map must be defined for the AGV to be able to localize itself. There are multiple sources that can initiate the driving, but this document mainly concentrates on the Navitrol Monitor software which is the main tool when taking new vehicles into use.

Different driving motions

By default, the vehicle follows the route with one continuous motion so that the heading of the vehicle follows the tangent of the route. This section provides a few alternatives for increased flexibility of the AGV motion.

Pivot point

Most of the AGVs can rotate around their navigation point which often is useful in compact spaces. In Navitec's products, these are called as pivot points and they can be added when defining the routes for the project in Navithor Tools. The vehicle will slow down before it stops at a pivot point, to then start rotating around its axis to the heading of the subsequent route.

Figure 2: Driving through a pivot point

Wheel turning point

Wheel turning points are special pivot points that are available for omnidirectional AGVs. The difference between a pivot point and a wheel turning point is that the AGV frame can't turn on a wheel turning point. The AGV stops on a wheel turning point. Turns the wheels to the driving direction and continues the when the wheels point to the correct direction.

Direction change

A direction change means that the sign of velocity in the route changes. A simple example is that vehicle drives first in forward direction, stops at the direction change point and then continues driving in reverse (see Figure 3). In similarity to pivot points, the vehicle will slow down before the direction change to stop more accurately. A short stop during the direction change is defined by the parameter pc_direction_change_wait_ms with the purpose to let the motor controllers stabilize before changing the direction. Vehicles which have steered wheels will also wait for the wheels to reach an acceptable angle before giving a speed reference.

Figure 3: Direction change motion

Omni driving

For vehicles with omni directional capabilities, where the frame heading is not tangent to the direction of motion, additional options are available. Route sections can be defined to be driven with a certain frame heading with respect to the map coordinates. For example, in Figure 4 the vehicle drives in and out of pivot point maintaining the same orientation during the whole motion. Only the wheels of the vehicle are steered 90 degrees on the pivot point to move the vehicle accordingly.

Figure 4: Omni driving in a pivot point. Vehicle maintains its orientation.
