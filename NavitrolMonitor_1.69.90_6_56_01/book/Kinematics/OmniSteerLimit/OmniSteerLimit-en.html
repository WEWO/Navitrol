Omni Steer Limit

Updated 26.01.2024

Lightly steer limited omni -- total range over 180 degrees

Introduction

Overview

Steer limit functionality for omni drives that do not have freely turning wheels because of physical limitation, such as connected wires.

Functionality

The omni steer limit limits steer angles of omni AGV between the given maximum and minimum angles.

1. When omni's steer wheel is approaching the steer limit AGV slows down

2. After certain limit is reached the AGV performs wheel adjustment

  • Performs soft stop

  • Turns wheels to the as middle of the total allowed angle range as possible

  • Resumes driving

Note 1: If the steer angle gets within 5 degrees of the minimum or maximum limit the AGV will perform a hard stop instead of soft stop.

Applicability

Omni steer limit can be used only with omni AGV systems where Navitrol has direct control over all steer limited steering wheels. Typical examples of such cases are AGVs with Schwarchmueller motor controllers.

Parameters

  • I, omni_steer_limit_enable[i]

    • Functionality

      • Enables/Disables the use of omni steer limit for defined wheel index

      • Other three parameters do not matter if this one is disabled

    • Values

      • Min: 0 - Omni steer limit disabled

      • Max: 1 - Omni steer limit enabled

      • Default: 0

  • F, omni_steer_limit_CCW_angle_deg[i]

    • Functionality

      • Defines the CounterClockWise hard limit for a single wheel

    • Values

      • Min: 0

      • Max: 179.9 (degrees)

      • Default: 179.9 (degrees)

    • F, omni_steer_limit_CW_angle_deg[i]

    • Functionality

      • Defines the ClockWise hard limit for a single wheel

    • Values

      • See omni_steer_limit_CCW_angle_deg[i]

  • F, omni_steer_limit_angle_tolerance_deg

    • Functionality

      • Defines the size of a buffer that triggers the wheel adjustment

      • Leaves some room for control so that the wheel can be adjusted before it reaches any hard limits

      • If the parameter is decreased below the default value, it is not guaranteed that the wheel never reaches the hard limits.

      • Bigger the buffer the smoother the stop to adjust the wheels will be.

      • At minimum value there will be only hard stop

      • Same value is used for all steer limited wheels

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 20

  • F, omni_steer_limit_swap_turn_tolerance_deg

    • Functionality

      • Determines how close to the swap target angle the wheel must be turned before wheel adjusting is considered as done

      • Smaller values are more reliable.

      • Larger values can be used for AGV:s with fast and accurate steering as they reduce some potentially redundant control.

    • Values (degrees)

      • Min: 0.1

      • Max: 90

      • Default: 2

  • F, omni_steer_limit_hard_stop_buffer_deg

    • Functionality

      • Determines how close to the min and max steer limits measured angle must be before activating hard stop

        • Min max limits defined by

          • omni_steer_limit_CCW_angle_deg

          • omni_steer_limit_CW_angle_deg

      • Activating hard stop can prevent the AGV from drifting out of route and losing position.

      • Should be adjusted only for machines with very limited total range as generally it would be better use larger omni_steer_limit_angle_tolerance_deg values so that AGV has enough time to perform slow stop.

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 5

  • F, omni_steer_limit_active_control_before_stopping_buffer_deg

    • Functionality

      • Determines how close to the min and max steer limits measured angle must be before the steer reference is repelled from the limit even if still moving

      • Using this parameter is a tradeoff for protecting the set limits and staying on the route as when the measured angle is over this limit is a strong steer reference is given away from the limit. This will cause the AGV lose position in most cases so this should be used only as last resort.

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 3

  • F, omni_steer_limit_adjust_additional_wheels_buffer_deg

    • Functionality

      • When one wheel has reached the limit. This value will be added to omni_steer_limit_angle_tolerance_deg to check if multiple wheels should be adjusted simultaneously during a single stop.

      • This will not be done if the check limit would go over the defined angle range.

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 20

  • F, omni_steer_limit_max_reference_diff_deg

    • Functionality

      • Determines how close to the min and max steer limits the reference can ever be.

      • Example: limit 180, value 1, max reference is 179

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 1

  • F, omni_steer_limit_soft_slow_buffer_deg

    • Functionality

      • This value will be added to omni_steer_limit_angle_tolerance_deg to slow down when nearing the limit.

      • This value could be increased when using very small total angle ranges (< 200 deg) as the buffer sizes have to be smaller so steer limit would be met at slower speed and hard stop and active adjustment limits would not be met.

        • The slow down target speed can be defined with parameter: pc_soft_slow_wheel_speed (default value 0.3m/s).

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 10

  • F, omni_steer_limit_max_control_difference_deg

    • Functionality

      • Determines the maximum difference between reference and measured angle.

      • The reference angle will never exceed this difference.

      • Example:

        • Navitrol would like to control a wheel to angle of 90 degrees.

        • Current angle is 0 degrees.

        • For any reason, the wheel does not follow reference and stays in zero degrees.

        • Reference angle will be increased based on mot_ctrl_max_steer_rate_deg_p_s until the defined control difference has been reached and then stays there.

        • With default value the reference would stay in 30 degrees until wheel starts to move again.

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 30

Recommended minimum total range for smooth performance is 270 degrees.

Minimum total range is 180 degrees in theory. (In practice > 200 degrees)

Total range = omni_steer_limit_CCW_angle_deg + omni_steer_limit_CW_angle_deg

Other parameters are recommended to be kept in their default values unless instructed otherwise by Navitec Systems.

Troubleshooting

  • Defined steer limit is exceeded

    1. Check that the limits are defined correctly

      • Right limits for each wheel

      • CCW and CW limits are not mixed

    2. Is CCW limit or CW limit over 180 degrees?

      • It is possible that the value stored into startinfo.bin is out of sync

        • Shutdown Navitrol

        • Turn all wheels manually to the zero angle

        • Remove the startinfo.bin

        • Restart Navitrol

    3. If steps above do not help

      • Take a log and contact Navitec support

  • AGV loses position when the steer limit is reached or stops too hard

    1. Increase value of omni_steer_limit_angle_tolerance_deg

    2. If steps above do not help

      • Take a log and contact Navitec support

Heavily steer limited omni -- total range less than 180 degrees

Introduction

Overview

Steer limit functionality for omni drives that are heavily limited. A heavily limited omni drive can only turn a wheel less than 180-degree range.

Functionality

The omni steer limit limits steer angles of omni AGV between the given maximum and minimum angles. It has two step protection:

1. In route generation

  • Paths with higher curvature than what given limits are discarded

  • ERROR_CURVATURE_TOO_HIGH is set

  • If this happens when giving drive orders on a defined path, the path must be manually adjusted

2. When calculating final references

  • With generated routes this is only failsafe. However, driving AGV manually will use only this.

  • If required curvature would exceed the allowed limits. Maximum curvature to same direction will be used.

    • WARNING_LIMITED_STEERING_ACTIVE is set to indicate this

  • Waning - when curvature has been limited the AGV will drift out of the route. If it limited only briefly the AGV will be able to recover. Otherwise AGV will eventually stop with ERROR_POS_CONT_POS_ERR

Applicability

Can be used with any omni drive. However, if Navitrol does not control the wheels directly, it is user's responsibility to follow the given references.

Parameters

  • I,omni_steer_limit_enable[i]

    • Functionality

      • Enables/Disables the use of omni steer limit for defined wheel index

      • Other parameters do not matter if this one is disabled

    • Value: 1 = enable

  • F,omni_steer_limit_CCW_angle_deg[i]

    • Functionality

      • Defines the CounterClockWise limit for a single wheel

    • Values

      • Min: 0

      • Max: 180

  • F,omni_steer_limit_CW_angle_deg[i]

    • Functionality

      • Defines the ClockWise limit for a single wheel

    • Values

      • See omni_steer_limit_CCW_angle_deg[i]

  • F,omni_steer_limit_angle_tolerance_deg

    • Functionality

      • Defines the maximum angle tolerance left when checking the trajectory

      • Leaves some room for correcting position error while driving

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 20

      • Recommended: 5

  • F, omni_steer_limit_max_reference_diff_deg

    • Functionality

      • Determines how close to the min and max steer limits the reference can ever be.

      • Example: limit 180, value 1, max reference is 179

      • Is used to activate limited steering

      • Used to prevent breaking the reference if the steering control might overshoot from the given references

    • Values (degrees)

      • Min: 0

      • Max: 90

      • Default: 1

  • F, omni_steer_limit_swap_turn_tolerance_deg

    • Functionality

      • Activates soft stop the difference between measured and reference values during wheel adjustment is larger than given value.

      • The parameter is used, and the soft stop can be activated only when the WARNING_LIMITED_STEERING_ACTIVE -warning is active.

      • The goal is to prevent driving before all wheels are adjusted right angles.

    • Values (degrees)

      • Min: 0.1

      • Max: 90

      • Default: 2
