NAVITHOR USER INSTRUCTIONS

Statistics

This document contains information about statistics handling in Navithor

Updated 04.02.2022

Introduction

There are 2 types of statistics generated by Navithor.

First type is old generation of statistics which can be found inside ProductionData directory. These statistics are always generated in a verbose format. These can be turned on or off by changing parameter StatisticsEnabled inside ConfigFiles/ServerSettings.json. Older statistics will continue to be generated but nothing new will be added.

Second type of statistics is configured differently and supports multiple output formats. It's possible to output statistics to:

  • Dedicated statistics database
  • Verbose format
  • CSV format

Server restart is required after changing configuration to enable or disable statistics generation for both type 1 and type 2.

Configuration

Statistics will be enabled when either:

  • Logging to database is enabled by parameter Statistics_UseStatisticsDatabase or
  • Logging to text files is enabled by parameter Statistics_UseFleetControlStatisticsTextLogger and at least one format is specified by Statistics_TextLoggerFormat.

Figure: Statistics parameters

Statistics database

If logging statistics to database is enabled then a separate Statistics database is required. By default, enabling statistics logging to database will generate a new database called NTDB_Statistics automatically after the server has been restarted. For manual configuration see * SQL Server and Database installation guide* - document for further instructions. Currently there is a technical limitation that the statistics database has to be on same SQL Server instance as main database.

Statistics view in the Web client will only appear when Statistics database is enabled.

Text format

Text format statistics can be enabled by changing Statistics_UseFleetControlStatisticsTextLogger to true and enabling one or more output formats on Statistics_TextLoggerFormat. Text format statistics can be found inside Logs\Production directory.

  • Text logs will be generated to Logs/Production directory
  • Each type of statistics data has it's own file
  • Each output format of statistics data is saved to it's own file
  • Once log file reaches maximum size it will be moved to Old directory
  • There will be a directory for each day inside Old directory
  • Files moved to Old directory will be prefixed with "yyyy_MM_dd_"

Considerations with Statistics database

Logging statistics to database will cause increased usage of SQL Server instance. In extreme cases this might slow down access to production database which in turn will slow down the entire Navithor system. Please ensure that the computer hosting the SQL Server instance is powerful enough.

Considerations with Text log statistics

This will cause more IO to be done on storage device and will require more storage space to be available for Navithor Server. Amount of extra storage space required depends on multiple factors: Amount of AGV's and production speed being biggest.
