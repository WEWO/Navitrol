Terminology

TermExplanation
UserA user in Navithor Fleet Control. Can be a Navithor user or Active Directory user.
Navithor UserUser in Navithor Fleet Control, managed by Navithor
Active Directory UserUser in Active Directory, managed by Active Directory instead of Navithor
UserRoleUser can belong to one or many Roles, each Role grants 0 to many Rights.
UserRightsRight to view or manage something. For example: "Give GOTO command" or "Edit server settings".

Introduction

Navithor Client allows users, for example, to change server configuration and give manual commands to AGV's. Some of the actions can cause production to halt and thus should be restricted to certain users only. The User roles allow restricting these potentially harmful actions.

Basic concept is that each action or view is behind a UserRight and if user is missing this UserRight then the view isn't visible or action is not allowed. These UserRights are granted to UserRole (for example: operator, administrator). The user can have multiple UserRoles.

Practical example:

Role nameActions allowed
ConfigurationAdminChange Server Settings, Server Parameters, AreaFiles and Machines
SeniorOperatorView waypoints, Change resource status, Give any command to AGV, Abort task, Enable/Disable production
JuniorOperatorAbort task, Give GOTO command, Give Load/Unload
UserAdminCreate user, Delete user, Modify User, Modify Active Directory mapping

User that has both SeniorOperator and ConfigurationAdmin would be able to:

Change Server Settings, Server Parameters, AreaFiles and Machines, View waypoints, Change resource status, Give any command to AGV, Abort task, Enable/Disable production.

Granting Admin Rights gives full access to system and should be given sparingly.

Difference between UserLevel (old) and UserRoles (new)

UserLevel

  • User is at single UserLevel
  • Each UserLevel has non-configurable Rights

UserRoles

  • Ability to create new Role with any name
  • Rights each Role grants are configurable
  • Better integration to ActiveDirectory
  • More fine-grained control

Default user roles

Role nameExplanation
AdministratorAdmin. Can do anything.
OperatorUnable to configure system. Can command AGV. Can change symbolic point state
Production ManagerCan view production. Unable to give commands to AGV. Unable to configure system. Unable to change symbolic point state.
GuestView main window, production orders, resources only. No access to other views or any configuration.

Refer to ** Default roles and user rights matrix** documentation for more details about available user Rights and default roles.

User management can be accessed by Production Settings -> User Management -> Users. Requires "Manage users" Rights.

Creating new user

Configuring Active Directory Mapping

Only users managed by Navithor can be directly added to UserRoles. Active Directory users and roles they belong to are managed by mapping Active Directory Group to Navithor UserRole.

Menu can be accessed by Production Settings -> User Management -> Active Directory. Requires "Configure Active Directory" Rights.

Navithor does not read possible Active Directory Groups and they need to be filled out manually. Adding new entry can be achieved by typing anything into empty textbox, this will also generate a new blank entry automatically.

Mapping Active Directory Group to NavithorRole

Active Directory mapping comes with preconfigured defaults, but these can be changed or deleted.

Active Directory GroupUserRoleNotes
AdministratorsAdminDefault name of admin Role on Active Directory, automatically mapped to Admin on Navithor.
GuestsGuestDefault guest users on Active Directory.
Domain UserGuestAny user in Active Directory.
OperatorsOperatorOperators group doesn't exist by default.
Production managersProduction managerProduction managers group doesn't exist by default.

More about Active Directory

  • Mapping a UserRole into non-existing Active Directory Group is allowed but doesn't do anything
  • Custom Active Directory Groups cannot be nested. They need to exist on top-level.
  • Logging with Active Directory is not enabled by default.
  • Navithor supports hybrid mode for Login. Users can always login with Navithor User credentials if needed. Active Directory users are automatically authenticated using currently logged in user in Windows.

Active Directory is covered more in detail in Using Active Directory for authentication document.
