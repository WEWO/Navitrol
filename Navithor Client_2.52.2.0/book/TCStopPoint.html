Safe stopping point calculation

This document contains information on safe stopping point calculation (TCStopPoint) in Navithor.

Updated 11.08.2021

Introduction

While Scheduler is planning a route for the machine, it is releasing the route segment by segment. One segment might be sometimes very long, and it is necessary that safe machine position can be defined more precisely. For this purpose there exists implementation for calculating safe stopping point.

The safe stopping point is calculated based on complex logic of ruleset. All different objects on the route will affect it - OneMachineZones, doors, other machines, etc. After combining all the rules together, the function returns one single point, the point how far the machine is allowed to advance on the route.

Fleet control stopping point calculation

When machine is driving on the field, it follows the route which has been pre-planned for it. Might be that driving must be sometimes interrupted by some external or internal trigger. External trigger can be, for example, a door tag or fire alarm. Internal trigger can be other machine crossing the route, or resource drops too close to path.

Fleet control handles these interjacent stops by looking for multiple different situations which might occur if machine continues driving its route. If Fleet control notices that machine cannot proceed after certain point, it marks this point on the route with a specific tag called stopping point. For each type of stop it is possible to define separate parameter which controls the distance (wait distance) how close to this point machine can drive. The point until machine can proceed, when wait distance parameter has already taken in the account, is called safe stopping point, or TC stop point. When machine reaches this point, is must stop and it cannot proceed until object has been removed or situation has been changed somehow.

In the following Figure the stopping point for "AssemblyOmni 1" in front of "AssemblyOmni 2" is visualized with an orange dot, and segments which are released for a machine to drive are shown with a green line. The machine can then proceed until the orange dot.

Stopping point types and parameters

Fleet control supports multiple different types of stopping point. By default, if there are no obstacles in the route, machine can drive until stop point called Target. Another final stop point is Rack.

If route is very long, Fleet control does not release all the segments on the route, instead machine can be proceed until MaxDist stopping point. This point will be moved always further away when machine drives.

Add Safety Distance parameter

In addition, there exists a parameter TrafficController_TC_SAFE_STOP_ADD_SAFETY_DIST_M which can be used to add extra safety marginal for each interjacent stopping point type. For specific stopping point types, like Door, Rack, MachinePosition, Target, TrafficLight, and MaxDist, this parameter does not affect.

Parameter can be also negative, when it reduces the wait distance, which is defined by other parameters. For example, machine stops before oneMachineZone (OMZ) and wait distance distance TrafficController_ONE_MACH_ZONE_WAIT_DIST is specified to 3 meters, and TrafficController_TC_SAFE_STOP_ADD_SAFETY_DIST_M is set to 1 meter, then wait distance in front of the OMZ is 2 meters.

Waypoint end

Scheduler_WAIT_DIST_FROM_WP_END_DEFAULT_BOUND defines the distance how close to the end of the waypoint the stopping point can be set. If any interjacent stopping point type is expected to be set closer to waypoint end, stopping point will be moved closer so that specified distance is reached.

Forbidden distance parameter

Safe stopping point calculation supports the forbidden area in front of the target point. It can be controlled with a parameter TrafficController_TC_SAFE_STOP_FORBIDDEN_DIST_FROM_TARGET_M, meaning that when this parameter is set over zero, safe stopping point will never be set closer to target than this specified distance. For Rack type of symbolic point, separate parameter Machine_DriveAwayFromTargetDistance should be used instead for the similar functionality.

One machine zones

One machine zones (OMZ), as name describes, are zones where only one machine is allowed to drive simultaneously. TCStopPoint functionality is taking care that machines do not drive inside the zone unintentionally. If machine is allowed to enter the zone, no stopping point is set before the zone. If zone is already occupied or reserved, stopping point type OneMachineZone will be set to the point where machine can proceed. Waiting distance is controlled by parameter TrafficController_ONE_MACH_ZONE_WAIT_DIST.

Multiple OneMachineZones can be also combined together. Below are some examples how several zones work together.

Overlapping zones

If two zones are overlapping and machine should stop inside the OMZ, the stopping point is moved earlier so that it is not inside the zone anymore.

Figure: Two zones overlapping.

Consecutive zones

If two zones are defined consecutive, but are not overlapping each other, stop point must be set inside the first zone, at the waiting distance from the second zone.

Figure: Consecutive zones.

OneMachineZone and Door after

If there is a door after the OneMachineZone, stopping inside the zone is allowed.

Figure: OMZ with Door tag after.

OneMachineZone and Door inside

If door is defined inside the one machine zone, and stopping point for door will be outside of the zone, stopping point for OMZ is used instead.

Figure: OMZ With Door inside the zone.
