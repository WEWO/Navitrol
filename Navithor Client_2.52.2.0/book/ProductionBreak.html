NAVITHOR USER INSTRUCTIONS

Production Break

This document contains information on how to use the Production Break feature with Navithor Fleet Control.

Updated 09.12.2022

Version History

VersionDateAuthorChange
1.018.11.2022Toni LiskiDocument created.
1.109.12.2022Simon ClementAdd warning about decision points in wait queues

Overview

The Production Break feature enables a more sophisticated way to control the longer breaks in the production, for instance, when the production is paused for the weekend. Once the Production Break mode is activated, the Fleet Control drives machines automatically to predefined locations and puts them in a sleep to save the battery.

The Production Break considers active production, so the machines which are carrying the load are first dropping off the load, before driving to the break locations. Also machines are required to be charged up to the specified level before driving to the break locations.

Sleep mode

The sleep mode is a special feature which any machine can support. Fleet Control will command machines to the sleep mode to save the battery during the production break, or sleep mode can be started manually from the Clients. The Sleep mode requires custom implementation on the supervisor interface (See Navitrol documentation for more details).

If the machine supports the sleep mode, the machine is commanded automatically to sleep once it reaches the production break location. When production break is over, machines are woken up automatically and they should continue the normal production. The production break logic does not require that machine supports sleep mode.

Navithor Tools

Defining the Production break locations

Enabling the production break feature requires that at least one symbolicpoint has a ProductionBreak type defined in Navithor Tools. Any pickup, dropoff, waitForPickup, or waitForDropoff location can be defined as a production break location. It means that when production break is activated, machines are allowed to use only those symbolicpoints for waiting.

It is also possible to define symbolicpoint only as ProductionBreak. Then this symbolicpoint is not allowed for the normal production, but only when the production break is activated.

Figure 1: Each symbolicpoint where waiting is allowed during the production break should be marked with a ProductionBreak type in Navithor Tools.

A typical use case would be to define a long wait queue on the one-way lane for production break purpose. This allows that all machines gather at the same, known places when break is activated, and are not scattered around the factory.

Figure 2: One typical use case is to define production break location as a wait queue.

Note: Do not place decision points inside a wait queue. It is not supported and may lead to unwanted behavior.

Limiting locations for specific machine types

Each symbolicpoint (production break location) can be defined to allow only specific machine types to drive to the location. The allowed machine types can be selected in Navithor Tools by selecting symbolicpoint and selecting types from the button Select Enabled MachineTypes.

Backup dropoff locations

When the production break is activated and a machine has a load aboard, there might be situations that there is a machine or a load blocking the target location, and the location will not be available until the production break is over again. For this kind of situations it is possible to define a backup dropoff location in Navithor Tools, which is allowed for certain type of the loads and certain type of the machines. The backup dropoff must be defined as PickupDropoff location, as both actions must be allowed on the location.

Defining the backup dropoff location is done from the Destinations And Loads menu, selecting Edit button under the Load Types. The opened dialog allows to specify certain load types for which the backup dropoff is allowed.

Figure 3: Defining location as a backup dropoff location in Navithor Tools.

Activating Production Break

The production break mode can be activated either from Navithor Client or over the MES interface (see message StartStopProductionBreak (ID = 359) in the MES interface documentation). When starting or stopping the break from the Navithor Client, the user responsible must have the role "Start or stop production break" granted.

A button to start a production break can be found from the main menu, Machines -> Start production break. Confirm with Yes button.

Figure 4: Production break can be started from Navithor Client.

Once production break is activated, the indicator Production on break can be seen on the bottom left corner of the map view.

Figure 5: Production break status is visualized on a bottom left corner of the map.

Charging rules

There are currently two system parameters affecting the charging logic with production break:

BatteryLevelToAllowDrivingToProductionBreakWithoutCharging

  • The minimum battery level an AGV needs to have in order to drive directly to production break location without charging. If battery level is below this minimum level, charging is required. After the AGV has started driving towards a production break location, charging request will not be generated even if the battery level goes below this limit.

BatteryLevelToReleaseAGVFromChargerToProductionBreak

  • The parameter to control what battery level an AGV needs to reach before it can be released from the charger to the production break location in case the production break is active.

These parameters override the normal battery levels defined in system. Even if the battery level would be high enough to execute normal production, it might be useful to set parameter BatteryLevelToAllowDrivingToProductionBreakWithoutCharging higher to force machines to charge a bit before entering the production break. While production break is active and machine is already at the production break location, the charging is never allowed, even if the battery level drops under the specified limits.

Note: During the production break, the rules defined for AlternativeChargingRulesZone are ignored.

Production rules when activating production break

After production break mode is activated, the rules below are followed:

Machine does not have a load aboard and is not currently picking up anything

  • Machine can drive directly to a production break location.

Machine hasn't picked the load yet

  • The production order is removed and machine drives to production break location. Unhandled orders (transfer requests and pickup requests) are still kept, so when production break is over, new production orders are generated based on those.

Machine has a load aboard

  • Machine drives to assigned dropoff location and completes the current order. After dropoff it can continue to a production break location.

Machine has load aboard, but target location is not available

  • Machine is allowed to use the backup dropoff location. See Section Using backup dropoff locations.

Machine is not in production

  • Automatic production break mode requires that machine is in production. Machines out of production stay where they are.

Machine is charging

  • Machine is released when battery level reaches the parameter BatteryLevelToReleaseAGVFromChargerToProductionBreak. See Section Charging rules.

Generally: Once the production break is activated, the active production orders are executed and completed if the machine has already picked up the load. If the load linked to the production order has not been picked yet, the production order is removed and machine drives to production break location.

Note: If production is executed only with Production orders (generated from over MES with AddProductionOrder message), they must be sent again once production break is over.

Using backup dropoff locations

If a machine has a load aboard, but target location is occupied by a load or a machine, the machine is allowed to use the backup dropoff locations. The machine drives the load to the backup dropoff location, and after dropping it off, the machine continues to the nearest production break location. A new transfer request is automatically generated to transfer the load from the backup dropoff location to the original dropoff location. When the production break is over, closest machine drives back to the backup dropoff location, picks the load and continues to the original dropoff target with the load.

Note: Machine cannot drive to production break location until it has dropped off the load. If all backup dropoff locations are reserved, the machine must wait.

Custom rules to distribute machines after the break is over

Once the production break is over, machines are distributed around the production area by assigning the available production orders for them automatically or by choosing the closest waiting location. There is also a way to define more sophisticated logic to spread the machines around the area by specifying a production break release priority for the waiting locations.

The special priority is entered in the file LocationPriorities.txt, which can be found under <server root>\Overrides\ folder. The format of the file is following:

<symbolicPointId>,<locationPriority>,<productionBreakReleasePriority>

For example:

# File for specifying the priorities of locations.
# Format: SymbolicPointId,Priority,ProductionBreakReleasedPriority
# Example: 3,2,4
14,4,1
17,4,3
18,4,4
21,4,1

which defines a custom production break release priority 3 for the location (17) and a priority 4 for the location (18).

After the production break is over, all the locations with ProductionBreakReleasedPriority higher than 1 are collected and location requests are generated for those locations. Machines will select the best location and fulfill the location requests one by one. The maximum amount of location requests that can be triggered is limited by amount of machines currently targeting the production break locations.
