Client Updater

This document introduces Client Updater feature

Updated 30.04.2020

Introduction

Client Updater is a feature which allows Navithor Server to send newest Client binaries automatically when outdated Client connects to Server. This streamlines updating Client binaries to newest version.

To take this Feature into use:

  • Create a Directory "ClientBinaries" in Navithor Server installation directory (if missing).
  • Copy a .zip file containing latest Navithor Client version in ClientBinaries directory.
  • This .zip should follow a specific naming convention: "NavithorClient_MAJOR.MINOR.BRANCH.REVISION.zip" or "NavithorClient_MAJOR_MINOR_BRANCH_REVISION.zip". For example: "NavithorClient_2.17.0.0.zip"
  • Next time Navithor Client connects to Server there will be a prompt to User about update.
  • It is required that both Navithor Server and Navithor Client have same version.

What happens if you connect to Server with outdated client

Case 1: If there are no binaries available then Server will inform Client. Outdated Client will not be able to connect to that Server. This situation requires either adding the matching Client binaries to correct directory or manually updating Client.

Case 2: Server finds the correct binary and sends it to Client. Client will notify user once download is completed. New version is downloaded to Client directory in .zip format. To take new Client into use simply unpack the .zip file to Client directory.
