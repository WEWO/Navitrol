Introduction

This manual gives insight into the Web Client interface and provides various use cases for interacting with the Web Client, carrying out interactions with the AGVs and production environments.

Web Client is a simplified version of Navithor Client. The main benefits of the Web Client are that it can be accessed with mobile and desktop browsers and it allows multiple simultaneous connections.

Web Client can be used to observe and control the production:

  • Observe

    • Map

    • Orders and their status

    • Errors and events

    • Machine states

    • Symbolic point states

  • Control

    • Take machines in and out of production

    • Initialize machine positions

    • Add and remove

      • Production orders

      • Resources

Logging to the Web Client

The Web Client can be accessed through any device having an internet connection and web browser.

Steps of connection:

  1. Open the web browser (Chrome or Firefox recommended)

  2. Type URL to the Web Client in the format, 'protocol://ip_address:port'

    a. The protocol in the URL is either "http" or "https" depending if the connection was configured to be secure (https) or not (http) during the installation of the Web Server.

    b. The ip_address is the IP address of the computer where the Navithor Server is running

    c. The port is the one that is configured while setting up the web server during the installation of the Navithor Server.

    d. A typical example of the URL would look like this: https://192.168.1.46:1234

The login page for Web Client prompts the user to enter username and password. (Note that users can only be added and removed using Navithor Client.)

Figure : Login Page

Web Client pages

All pages can be accessed from the top bar of the website when using the Web Client on large windows and on small windows such as mobile screens pages can be accessed from a drop-down menu in the top right corner of the view.

Note that Web Client has been translated to German and the language options are located under the "options" icon:

Overview page

The overview page (figure Overview page) provides an overview of the production status of AGV fleet, amount of active production orders, errors and warnings. On the right-hand side of the interface list of events can be seen with date and time. More information of any of these categories can be found accessing the related pages.

Figure : Overview page

Map page

The Map-page displays the map of the production environment consisting of all the routes, resources and AGVs.

An AGV can be selected to open an option window (figure Map and an option window (on the left) to manage an AGV) which can be used to directly control the AGV, manage resources on it etc. More information about controlling an AGV can be found in the chapter "Controlling AGVs".

Symbolic point can be accessed in similar manner to modify resources on them or enabling/ disabling them from a production.

Figure : Map and an option window (on the left) to manage an AGV

Pickup orders

Pickup orders can be made by selecting a load symbolic point with resources and pressing the REQUEST PICKUP button. See subsection "Requesting pickup".

Controlling symbolic points

Symbolic points can be controlled in similar manner with the AGVs.
First, select a symbolic point (for example from the Map page) to open editing view. This view shows the controls and current status of the selected symbolic point including name, MES relation, resources and active production orders.

Figure : Symbolic point editing view

Disabling a symbolic point

DISABLE button can be used to disable a symbolic point from production. Note that even when a symbolic point is out of production AGVs can drive there if the drive order is given manually.

Editing resources

"EDIT RESOURCES" button lets the user to add, remove and edit resources on the symbolic point. Note that this button is available only for Load, Unload and Load/Unload symbolic points.

Requesting pickup

Unhandled orders can be created using the REQUEST PICKUP button. Note that this button is available only for Load, Unload and Load/Unload symbolic points and it is disabled if there are currently no resources.

Respond to MES - option

Respond to MES checkbox determines if the symbolic point can be controlled using the MES-interface or not.

Controlling AGVs

Selecting an AGV from the Map page opens AGV control window (figure Map-page with AGV control window open on the left).

Figure : Map-page with AGV control window open on the left

FSTOP

The FSTOP (Also known as "fast stop") button will stop the selected AGV and generates a message on top that says 'Release FSTOP command sent' as seen in figure FSTOP active. After this the AGV can take no further actions until the FSTOP is released.

Figure : FSTOP active

The AGV can be released from the FSTOP using the RELEASE FSTOP button. Pressing the button generates a message 'Release FSTOP command sent' and releases the AGV from the FSTOP (figure Release FSTOP). After this the AGV is free to be taken into production or to be controlled manually.

Figure : Release FSTOP

Aborting Tasks

ABORT TASK button will stop the AGV and discard the task it was performing. Successful abort is confirmed with the message 'Abort command sent' on the top of the screen.

Request charging

REQUEST CHARGING button sends a request to the server to take the selected AGV to charging.

Release hold

RELEASE HOLD button can be used to release the AGV from a hold symbolic point to its next destination.

Taking the AGV in and out of production

An AGV can be taken in and out of production using the In Production switch (figure AGV out of production (left). AGV in production (right).). Note that taking an AGV into production will hide most of the manual control options.

Figure : AGV out of production (left). AGV in production (right).

Note that some controls are only available when the AGV is out of production.

Managing resources on AGV

EDIT RESOURCES button lets the user to edit or remove resources on the selecting AGV or add them if no resources are currently onboard (figure Adding a New Resource).

Figure : Adding a New Resource

The user can select the resource from the drop-down list and pressing SAVE RESOURCES button saves the resource on the AGV as seen in the picture below.

Figure : Resource Added

The LOAD-button has four states:

  1. START LOAD

  2. LOADING: ABORT

  3. START UNLOAD

  4. UNLOADING: ABORT

The state is determined based on if the machine has an resource or not, or if it is currently loading or unloading.

The button can be used on load and/or unload symbolic point to manually order the AGV to load/unload a resource on that point. If the button is used, when there are not such points nearby. Resource is generated on or discarded from the AGV. Examples are presented in figure Resource Loaded and figure Start Unload Button.

Figure : Resource Loaded

Pressing the START LOAD button loads the resource on the AGV from the point 55.

Figure : Start Unload Button

Pressing the START UNLOAD button will unload the resource that was previously loaded onto the AGV at the location 55.

GOTO POSITION

Pressing the GOTO POSITION button can be used to manually give a drive order to an AGV to move it to an arbitrary point. It prompts user to enter the destination. This can be done by clicking on the map or typing the symbolic point name into the text field (figure GOTO Position example).Once the destination is selected, pressing the GO TO SELECTED POSITION button starts the drive.

Figure : GOTO Position example

AGV driving to its destination point 55 can be seen in the figure GOTO Position example. The path chosen by the AGV to drive to its destination location is highlighted in green (current drive segments) and blue (whole path).

Figure : AGV Driving to Destination

Initializing Position

The position of AGV can be initialized using the SET POSITION button. The scanner measurements are rendered in green (figure AGV Driving to Destination) and the AGV should be positioned in such manner that those measurements match the environment.

Position initialization can be done by:

  • manually dragging the AGV to the right position;
  • selecting a symbolic point that has position initialization enabled;
  • clicking anywhere on the map.

Use rotation handle to adjust the heading of AGV.

Figure : Initializing AGV Position

Disabling and Enabling Machines

If a certain AGV is not in use, it can be removed from the production map by disabling it.
An AGV can be disabled using DISABLE button. It opens a small window that asks the user to confirm disabling the machine as seen in the figure AGV Driving to Destination.

Figure : Disabling AGV

Pressing the CONFIRM button removes the AGV from the map.

The AGV can be enabled again from the Fleet page's Disabled machines section.
Hovering the AGV on the disabled machine makes the ENABLE MACHINE button visible.
The button will enable the AGV once again.
Pressing the enable machine button will enable the machine back in the production map.

Return to Route

Return to route feature can be used when an AGV is a bit off the route. It generates a path back to the nearest route so it can resume production.
This can be done by pressing the RETURN TO ROUTE button.

Waypoints

The map contains a layer that allows the user to see waypoints and their state. Map displays by default forbidden waypoints only.

To show all waypoints, open map settings and enable "Show all waypoints" switch. If the switch button is not visible, user does not have enough rights to view all waypoints.

>Figure: Map settings and "Show all waypoints" switch

Waypoint can be set free or forbidden.

To change the waypoint state select a waypoint by clicking it and in the opened waypoint control window press button SET FREE or SET FORBIDDEN. If a button is not visible, user does not have enough rights to change the state of waypoints.

>Figure: Waypoint control window and selected waypoint status

Fleet page

The AGV fleet page can be accessed by selecting the Fleet.
As seen in the figure "Fleet page" the page displays information regarding all the AGVs present in the fleet. From here it is possible to take the whole group of enabled AGVs into production and to stop the production, by using START PRODUCTION and STOP PRODUCTION buttons. The view contains basic information of each vehicle such as the name, its type, the task it is carrying out and its current state.
Clicking the name of an AGV takes the user to Map page and shows the selected machine.

Additionally, the disabled machines can be enabled here -> Hovering a mouse over a disabled machine row reveals an "Enable machine"-button.

Figure : Fleet page

Locations

The Locations / Production locations page (figure Production locations), displays a table of all production locations present in the current production environment. The table can be sorted and filtered for easier access. Additionally, the name of a point can be clicked to view it on the map page.

Figure : Production locations

Orders

Orders / Production Orders page , displays a list of production orders in the system. Orders are divided into two tables: database orders and transfer requests in one table, and pickup requests for resources in other table. Following figure shows the information present in tables.

Additionally, there is two more options that allows an user to make new production orders and a new transfer requests. Making these orders is explained further in the next chapter.

Figure : Production orders page

Creating and editing orders and request

Production orders and transfer requests can be created and edited in "Production Orders" page. However, unhandled orders can only be created from the "Map" page.

The orders of any type can be deleted by selecting the Edit button and then pressing DELETE button.

Production Orders

To make a new production order, navigate to the production orders interface. At the production orders pressing the blue button titled as NEW PROD. ORDER opens the production order creation window.

Figure : Production orders view with the "New Production Order" button highlighted

Figure : Creation form for production orders - Unfilled on the left and a filled one on the right.

The 'Source' is the symbolic point containing a resource. Selecting the source from a drop-down list will allow the user to see which resource(s) are currently on that symbolic point. In figure Production Order Status the resource is named 'Pallet'. If the 'Source' location does not have any resource currently, it is not possible to create a production order from this location.

In addition to the 'Source' location and the resouce to be picked up, user can select the end destination of the order called as 'Target', using a drop-down list. The drop-down list shows all valid destinations that can be either single symbolic points or symbolic point groups. The name also indicates if the symbolic point belongs to a group. Replacing existing resource at the 'Source'

It is possible to allow replacing the resource type at the 'Source' location by modifying json_storage.json file. AllowReplacingCurrentLoadAtPickup key should be added under OrdersView and the value set to true as shown below:

{
  ...
    "OrdersView": {
      "Active": true,
      "AllowReplacingCurrentLoadAtPickup": true
    },
   ...
}

When this setting is enabled, the dropdown list displaying all resource(s) on the 'Source' symbolic point displays instead all allowed resource types for this symbolic point as a dropdown menu regardless of what the current resource is at the symbolic point. If the order is sent with a resource type that is different than the current resource type, Server will replace the resource with a new resource that has the correct type and the selected AGV can normally execute the order.

Even if this setting is enabled, it is not possible to create a production order from 'Source' location that does not currently have any resource.

Alarms

Alarms option on the page selection opens the 'Alarms' interface (figure Alarms page) that presents a list of all errors and warnings that are currently active. These errors and warnings can be sorted based on error level, source, origin, time of occurrence or description.

Figure : Alarms page

Alarms page has also filters to show only errors and warnings that have selected error level or are related only to a selected machine. There is also an option to filter alarms based on a text search.

It is possible to optionally hide or display fields Date occurred, Time occurred and Duration from parameters on the server side using json_storage.json file. Add VisibleAlarmsTableFields key inside AlarmsView key in the following format:

{
  ...
    "AlarmsView": {
      "VisibleAlarmsTableFields": {
        "Date": true,
        "Time": true,
        "Duration": true
     },
    "Active": true
  },
   ...
}

AGV availability statistics

AGV availability statistics chart shows the percentage of time AGVs spend in the error state during orders execution.

Standard statistics filters are applied to database records used for availability calculation, such as filter by time frame, AGV, source and target locations. In addition, availability can be adjusted to include/exclude certain alarms and time AGVs wait before order execution.

Configuring availability calculation

Availability calculation parameters can be preconfigured on the server side using json_storage.json file. Add StatisticsView key to the file in the following format:

{
  ...
	  "StatisticsView": {
    	"ShowAvailabilitySettings": true,
	  "IncludeWaitingTime": false,
	  "AlarmsExcludedFromAvailabilityCalculation": [1, 2, 3],
	  "MinimumAlarmDurationSec": 2,
	  "HighlightInGreenThreshold": 98,
    },
   ...
}
ParameterTypePurpose
ShowAvailabilitySettingsbooleanShow or hide Availability parameters section on the AGV availability page. If hidden, users will not be able to include or exclude waiting for execution time and alarms from the calculation
IncludeWaitingTimebooleanInclude or exclude time AGVs wait before order execution from the total orders execution time calculation
AlarmsExcludedFromAvailabilityCalculationarrayIDs of alarms to exclude from the total alarms active time calculation
MinimumAlarmDurationSecintegerThe minimum time in seconds an alarm has to be active to include it in it the total alarms active time calculation
HighlightInGreenThresholdintegerThreshold to highlight availability percentage in green color

Web Client Requirements

The Navithor Web Client has the need for a 'recent' web browser as it uses some of the latest web technologies.

Requirements for the use of the Navithor Web Client:

  • Supported Operating Systems:

    • Linux
    • Windows 7, 8, 10
    • Mac OS 10.15 +
    • Android 8 +

  • Supported Browsers:

    • Google Chrome 80 +
    • Firefox 68 +
    • Microsoft Edge 90 +

  • Minimum Hardware Requirements:

    • 2 Gig of RAM
      • Bigger production area will require more available RAM
    • 1.2 Ghz dual core CPU

Older operating systems like Windows CE, Windows Embedded CE 6.0, Windows Embedded Compact 7 are not supported.

No version of Internet Explorer is supported.

Web Client Login timeout settings

For the use cases where operators have the need to use the Web Client for long periods of time we can change the login timeout in order to allow the instances to remain active for longer without the need for re-logging in.

To access the setting we go to the "Settings" menu and then to the "Server Settings" tab.

Figure : Configuration menu

The value is in hours and it can be set between minimum 1 hour to maximum 87000 hours.

Figure : Timeout setting in menu

After saving the value the Navithor Server needs to be restarted for the setting to applied.
