Server as a service

This document contains information on how to configure Navithor Server to run as a service.

Prerequisites

  1. Navithor Server version 1.9.57 or later.

  2. SQL Server and database installation done.

  3. Navithor Server is configured and can be started.

  4. Running as a service is enabled by setting ConfigFiles/ServerSettings.json option RunAsService to true.

Important notes

  • Windows services listing can be opened by typing "services" to Windows search.

  • User account can be changed and configured in Windows services listing. If user account is changed, make sure that the new user has permissions to use the database if integrated security is in use.

  • If you want to temporarily run Navithor Server with the GUI without uninstalling the service, you can do this by changing RunAsService setting to false in ConfigFiles/ServerSettings.json file and starting the Navithor Server as normal application.

  • It is recommended that the full file path for Navithor Server.exe file and related .json file should not contain any whitespaces because sometimes it can prevent the installation from starting and leads to a regular start of the Server with the GUI.

  • Before moving the server to another folder the previously installed service must be uninstalled from the same directory where it was installed from.

Installing the service

  1. Open command prompt using "Run as administrator" option.

  2. Navigate to the directory where Navithor Server.exe is located.
    Note that this step is important as we write the execution directory to registry.

  3. Run command "Navithor Server.exe" -install

  4. Fill in the user account that is responsible for running Navithor. This should be the system credentials you use when log in to your Windows PC. To get the username, start command prompt and run command whoami. Use the result as the username and give the correct password for that user account in the installation prompt window.

    • Account name needs to be filled in the format: DOMAIN\UserName and if your computer does not belong to a domain, use *\UserName format instead.

  5. Check that FleetControlService is now listed in the Windows services listing.
    a. Services listing can be opened by typing "services" to the windows search.
    b. If FleetControlService is not listed:

    • Check EventViewer for errors.
    • See .InstallLog file from Navithor Server.exe's directory.

  6. Navithor Server can now be started from the Windows services listing. If service named FleetControlService is not in the listing, check the Troubleshooting section for more information.

Optional: User account running the service can be changed after installation. Navigate to Services -> Properties -> Log on to configure. This can also be a local system account instead of user account.

Granting permissions for SQL login

If you start Navithor Server normally it will connect to the preconfigured database with the currently logged in Windows user. However, the account used by a service also has to have SQL login with permissions configured on SQL Server. Depending on how SQL Server was configured there might be a need to grant additional permissions manually.

If FleetControlService is running but Navithor Client fails to connect after several attempts then follow the next steps to configure your SQL login:

  1. Open SQL Server Management Studio and connect to SQL Server using account with sufficient rights for SQL logins management. For example, an account with securityadmin or sysadmin role.

  2. Navigate to your SQL login under Security -> Logins:

  • If you use Local System account (in Services -> Properties -> Log on) then navigate to NT AUTHORITY\SYSTEM SQL login.

  • If you use Windows account then you need to create an SQL login with Windows authentication mode for this account first. For more information see the document * SQL Server and Database installation guide*, namely "Creating an SQL login for Windows authentication mode" chapter.

  1. Double-click on your SQL login and navigate to User Mapping page.

  2. Check Map checkbox in the upper part and grant db_owner role in the lower part to both NTDB_Production and NTDB_Statistics databases.

  1. Press OK button to apply granting permissions.

Uninstalling the service

  1. Open command prompt using "Run as administrator" option.

  2. Navigate to the directory in which Navithor Server.exe is located.

  3. Run command "Navithor Server.exe" -uninstall

Note: If the executable is not available anymore, to remove the service from the listing, open Command line prompt using "Run as administrator" option and run command sc delete MCSServerAsService.

Troubleshooting

ProblemPotential reasonSolution
Navithor Client can not connect to serverFleetControlService is not runningNavigate to Windows Services listing. Check if FleetControlService is in the listing and in the Running status. If status is not Running, right-click FleetControlService in the listing and click Start in the context menu
FleetControlService is not in the list of servicesFleetControlService is not installedGo to Installing the service chapter and perform the installation as described
FleetControlService can not start with Error 2: The system cannot find the file specifiedNavithor Server executable was moved or deletedRight-click FleetControlService in the listing and click Properties in the context menu. Check Path to executable value, it points to a location from which the executable is trying to start. Move executable files to the right location or re-install Navithor Server service (uninstall and install it again).
FleetControlService can not start with Error 1068: The dependency service or group failed to startLog On account issuesRight-click FleetControlService in the listing and click Properties in the context menu. Navigate to Log On tab, check Local System account option and press OK
FleetControlService is running, but Navithor Client can not connect to ServerDatabase connection issuesStop FleetControlService through Services listing and if needed kill Navithor Server process.

Navigate to ConfigFiles folder in Navithor Server executable directory, set RunAsService setting to false in ServerSettings.json file and run Navithor Server.exe. If needed, set up a new database connection.

Check if Client now is able to connect to the Server. If it is not, check NavithorLog.txt and startInfo.log for further troubleshooting.

When the problem is solved, shut down Navithor Server executable. Set RunAsService setting back to true and start FleetControlService from Services listing
